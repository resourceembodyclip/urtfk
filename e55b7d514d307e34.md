# WAP百科镜像站 · 结构化知识库链接聚合系统

项目名： Wapedia Link Bridge

Wapedia Link Bridge 是一个面向移动端知识图谱构建与历史内容回溯的技术资源聚合系统。该项目以 wap.baike.zr54.cn 域下的海量百科类文章链接为数据基底，提供结构化的链接索引、分类归并与快速检索能力，适用于内容迁移、语料采集、知识库初始化等多种工程场景。

本项目定位于技术运维人员、数据工程师、NLP 语料构建者以及开源知识库维护者。通过将分散的移动端百科条目链接统一收束为可维护的链接清单，配合自动化脚本实现批量可用性检测、内容抓取策略配置与变更监控，有效解决移动端百科内容碎片化、链接失效不可知、人工整理效率低下等问题。

## 功能概览

**批量链接状态检测**：对收录的全部百科文章链接执行 HTTP 状态码检查，自动标记异常链接并生成可视化报告。

**文章元信息提取**：从链接路径中解析条目 ID 与文件扩展名，支持按 ID 区间、数字特征进行筛选与排序。

**分类标签体系生成**：基于链接 ID 的哈希分布与数字段特征，自动为每个条目打上初步的分类标签，便于后续人工审核。

**增量更新与变更日志**：支持周期性拉取新链接，与已有清单进行差异比对，输出新增、删除、变更的详细记录。

**静态镜像生成器**：将链接清单与可用的条目元数据结合，生成轻量级静态 HTML 目录页面，便于内网离线浏览。

**导出与对接适配器**：提供 JSON、CSV、Markdown 表格等多种导出格式，可直接对接 Elasticsearch、Haystack 等检索与知识库系统。

## 应用场景

移动端百科内容迁移与备份：运维团队可利用本项目的链接清单，结合 wget 或自定义爬虫策略，批量下载指定条目内容，构建本地离线镜像，防止源站内容下架或访问受限。

中文百科语料库构建：NLP 研究人员可通过项目提供的结构化链接索引，按条目类型或 ID 区间采样，定向获取特定领域的百科文本，用于预训练语料扩充或领域词典构建。

百科条目变更监控与通知：内容审核团队可配置周期检测任务，当特定条目的链接状态或内容长度发生显著变化时，系统自动输出报告，便于及时复核。

知识库初始化与数据填充：企业级知识管理系统部署时，管理员可利用本项目的导出功能，快速将百科条目外部链接整合进内部知识图谱，作为冷启动阶段的外部知识源。

## 快速开始

以下操作基于 Linux / macOS 环境，确保已安装 Git、Python 3.9 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/wapedia-link-bridge.git
cd wapedia-link-bridge

# 安装依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 执行链接状态检测示例
python bin/check_links.py --input data/links_batch_29.txt --output reports/status_batch_29.json
```

## 项目结构

```

├── bin/                                # 可执行脚本目录
│   ├── check_links.py                  # 主检测脚本，接受 --input 和 --output

│   └── schema.yaml                     # 链接清单的数据结构定义

│   ├── operations.md                   # 生产环境运维指南
│   ├── development.md                  # 二次开发与扩展说明
│   └── data_schema.md                  # 输入输出数据格式详细说明
├── reports/                            # 检测报告输出目录
│   └── status_batch_29.json            # 示例输出：含状态码、响应时间、内容长度
├── src/                                # 核心源码目录
│   ├── fetcher.py                      # 封装 requests 的异步获取逻辑
│   ├── parser.py                       # 链接解析与 ID 提取函数
│   ├── reporter.py                     # 报告生成与格式化
│   └── utils.py                        # 日志、文件读写、哈希计算等工具函数
├── tests/                              # 单元测试目录
│   ├── test_fetcher.py                 # fetcher 模块的测试用例

└── requirements.txt                    # Python 依赖列表，含版本约束
```

## 常见问题

**Q: 检测脚本执行时出现大量连接超时，应如何优化？**

A: 可在 conf/default.yaml 中调整 timeout 参数（单位秒），并适当降低 concurrency 并发数以避免源站限流。对于持续不可达的链接，系统会自动记录并在报告中标记为 FAILED，建议后续通过人工复核或更换网络环境重试。

**Q: 如何只检测新增链接而不是全量重新扫描？**

A: 使用 bin/watch_update.py 脚本并指定 --since 参数，配合上一次检测的时间戳，系统会自动读取 data/ 目录下的历史清单进行差异对比，仅对新出现或 ID 发生变化的链接执行检测，大幅缩短执行时间。

**Q: 导出格式是否支持自定义字段？**

A: 支持。在 conf/schema.yaml 中定义所需的字段映射关系，然后调用 export_json.py --fields id,status,length 即可按指定字段输出。若需增加新字段，需同步修改 src/parser.py 中的解析逻辑。
