# WebLink Navigator

WebLink Navigator 是一个面向技术研究者、内容聚合者和知识管理从业者的结构化外链资源索引系统。该项目不生产原始内容，而是对分散在互联网各处的优质技术百科条目、开发文档和工程实践记录进行系统性收集、分类与元数据标注，解决开发者在信息检索过程中面临的资源碎片化、入口不统一以及检索路径过长等问题。

项目定位为轻量级外链门户，适用于个人知识库搭建、团队技术栈文档聚合、以及自动化信息采集管道的前端资源层。通过维持统一的条目格式和稳定的根域名映射关系，WebLink Navigator 能够在无需频繁更新核心代码的前提下，持续扩展其资源覆盖范围。

## 功能概览

**统一条目索引**：将所有收录的外链资源映射为具有唯一标识符的条目记录，支持按来源域名、内容类型和更新批次进行筛选。

**多维度分类标签**：每条资源自动关联技术领域、内容形态和目标读者层级三类标签，便于在不同使用场景下快速定位。

**元数据提取辅助**：基于 URL 路径结构与数字标识规律，自动推断条目的内容类型和可能的语言属性，降低人工整理成本。

**批次管理机制**：针对大规模资源导入场景设计批次编号系统，当前批次 35/56 提供完整的状态追踪和回滚能力。

**纯静态资源层**：所有索引数据可导出为结构化文档或 JSON 格式，无需数据库依赖，兼容绝大多数静态站点托管环境。

**条目级去重检测**：基于完整 URL 的哈希值进行重复校验，防止同一资源被多次收录导致索引膨胀。

**外链健康检查接口**：提供可插拔的链接可达性验证模块，支持定时检测失效条目并生成报告。

**快速跳转辅助**：生成可直接用于浏览器书签导入的 HTML 摘要文件，减少手动录入成本。

## 应用场景

技术团队内部文档站的外链聚合层。当团队拥有多个独立维护的组件库、API 文档和设计规范仓库时，可使用 WebLink Navigator 将所有对外引用链接统一收口至单一索引表中，便于定期审查链接有效性并统一更新策略。

个人知识管理系统的数据采集前置节点。知识管理从业者在构建个人维基或数字花园时，往往需要从大量临时收藏的网页中提取结构化信息。本项目的批次化导入能力可将这批原始 URL 快速转化为待处理队列，配合后续自动化标注流程。

自动化信息采集管道的资源清单维护。在搭建爬虫或 API 调用链路时，需要维护一份稳定的初始 URL 列表。WebLink Navigator 的纯文本索引格式支持直接被各类脚本读取，避免硬编码分散在多处代码中。

离线文档捆绑包的构建基础。对于需要在内网环境或隔离网络中部署技术资料站点的场景，可依据本索引列表批量下载对应页面内容，形成完整的离线快照。

## 快速开始

以下步骤可在 Linux 或 macOS 环境中完成项目初始化。

```bash
# 克隆仓库至本地
git clone https://github.com/weblink-navigator/weblink-navigator.git
cd weblink-navigator

# 安装依赖（需要 Python 3.8+ 及 pip）
pip install -r requirements.txt

# 运行索引构建脚本，生成当前批次的资源目录
python build_index.py --batch 35 --output ./output/batch_35.md
```

执行上述命令后，项目将在 output 目录下生成包含全部 180 个链接的 Markdown 索引文件，同时生成对应的 JSON 元数据缓存。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.8 或更高版本 | 是 | 用于运行索引构建、校验和导出脚本 |
| pip 21.0+ | 是 | 管理 Python 包依赖，包括 requests 和 click |
| requests 2.28+ | 是 | 用于可选的链接健康检查功能 |
| click 8.0+ | 是 | 命令行接口框架，提供子命令解析 |
| Git 2.25+ | 否 | 仅在通过源码安装或贡献代码时需要 |
| pytest 7.0+ | 否 | 仅在运行单元测试套件时需要 |
| markdown 3.4+ | 否 | 仅在导出 HTML 摘要格式时需要 |
| 网络连接 | 否 | 不强制，仅在执行健康检查时按需使用 |

## 项目结构

```
weblink-navigator/
├── build_index.py               # 主构建脚本，处理批次参数和输出格式
├── requirements.txt             # Python 依赖清单，锁定 requests 与 click 版本
├── config/
│   ├── base.yaml                # 基础配置：默认输出路径、日志级别
│   └── batch_35.yaml            # 当前批次的覆盖配置（来源域名白名单）
├── core/
│   ├── parser.py                # URL 解析器，提取数字ID和文件扩展名
│   ├── validator.py             # 链接格式校验与去重逻辑
│   └── exporter.py              # 导出模块，支持 Markdown / JSON / HTML
├── tests/
│   ├── test_parser.py           # 针对各类畸形URL的解析单元测试
│   └── test_validator.py        # 去重和健康检查的测试用例
├── docs/
│   ├── quick-start.md           # 五分钟上手教程
│   ├── batch-operations.md      # 多批次合并与冲突解决指南
│   ├── custom-parsers.md        # 扩展新域名解析规则的开发文档
│   └── architecture.md          # 整体设计说明与数据流图
├── output/                      # 构建产物默认输出目录（gitignore）
│   ├── batch_35.md              # 当前批次的完整索引文件
│   └── batch_35.json            # 结构化元数据缓存
└── scripts/
    ├── check_health.py          # 独立健康检查脚本，可定时运行
    └── import_legacy.py         # 导入旧批次数据的迁移工具
```

## 贡献指南

1. 在 GitHub 上 fork 本仓库，并克隆至本地开发环境。确保本地 Python 版本符合安装要求，同时安装所有开发依赖（pip install -r requirements-dev.txt）。

2. 新增资源条目时，请遵循现有的 URL 记录格式，并运行 build_index.py --validate-only 进行本地校验。若需要批量导入外部数据源，请参考 docs/custom-parsers.md 编写对应的适配器。

3. 提交变更前执行完整的单元测试套件（pytest tests/），确保 parser 和 validator 模块的通过率为 100%。对于涉及输出格式的变更，请额外检查 output 目录下的样例文件是否符合预期。

4. 发起 Pull Request 时请附带简要的变更说明，并在 PR 描述中注明本次操作涉及的批次编号和资源数量。若新增了配置文件字段或修改了导出结构，请同步更新 docs/architecture.md 中的对应章节。

5. 对于链接失效或域名迁移的报告，请通过 Issue 提交，并附上原始 URL 及可替代的新地址。维护者将定期合并此类修复并发布补丁版本。

## 常见问题

问：导入大量 URL 时出现重复条目如何处理？
答：核心模块中的 validator 组件会自动检测重复。若需强制覆盖，可在 build_index.py 中传入 --force 参数。对于历史批次中的重复数据，建议使用 scripts/import_legacy.py 执行迁移时启用 --dedup 选项。

问：是否可以自定义输出格式，例如生成 CSV 或 XML？
答：当前版本已内置 Markdown 和 JSON 两种导出格式。若需其他格式，可在 core/exporter.py 中继承 BaseExporter 类并实现 format 方法。社区已提供 CSV 扩展示例，位于 docs/custom-parsers.md 末尾。

问：健康检查脚本如何在内网环境中使用？
答：scripts/check_health.py 默认依赖外部网络访问。在内网部署时，可修改 config/base.yaml 中的 timeout 参数并配置代理环境变量（HTTP_PROXY / HTTPS_PROXY）。若完全离线，建议跳过健康检查步骤，仅使用 --skip-health 参数运行构建流程。
