# LinkVault Core

LinkVault Core 是一个面向技术团队与内容策展人的轻量级外链资源聚合与导航系统。该项目定位于解决分散在文档、笔记、书签栏中的海量外部链接难以统一管理、分类检索与版本追踪的问题，适用于需要高频引用外部技术资料、百科条目、规范文档或社区讨论的研发与知识管理场景。LinkVault Core 不提供爬虫或自动采集功能，而是通过结构化存储与标记体系，帮助用户将零散 URL 转化为可复用、可共享、可审计的知识资产。

## 功能概览

**批量链接入库与自动规范化** 支持从文本文件、CSV 或剪贴板批量导入 URL 列表，自动去除重复项并检测失效链接。

**多维度标签与分类体系** 允许为每条链接自定义标签、归属项目、优先级与阅读状态，支持层级分类与快速筛选。

**全文元数据提取** 对导入链接自动发起 HEAD 请求与可选的 HTML 标题提取，填充资源标题、内容类型与最后修改时间。

**链接健康度监控** 定时检查已入库链接的可访问性与响应码变化，标记 404、超时或重定向异常，生成健康报告。

**全文检索与高级过滤** 基于 SQLite FTS5 或 PostgreSQL 提供标题、标签、备注及 URL 片段的快速检索，支持多条件组合过滤。

**数据导入导出与标准化** 支持 JSON、YAML 及 Markdown 表格格式的批量导出，方便迁移至静态站点生成器或文档平台。

**访问审计与操作日志** 记录每一条链接的入库人、入库时间、修改历史与检查结果，满足团队协作与合规追溯需求。

**RESTful API 与 CLI 工具** 提供完整的 HTTP API 用于第三方集成，同时内置命令行交互界面，适配自动化脚本与容器化部署。

## 应用场景

**技术文档库的外链治理** 研发团队在维护 API 文档、架构说明或故障排查手册时，可将引用的外部规范、社区讨论或依赖库主页统一托管至 LinkVault Core，避免文档中散落大量裸 URL，同时定期自动检查引用链接是否仍有效，保障文档质量。

**开源项目 README 与官网的资源索引** 开源项目维护者可使用 LinkVault Core 管理项目主页、贡献指南、社区论坛及版本发布说明中的外部链接集合，通过标签区分“官方文档”“第三方教程”“视频演示”等类型，并在每次发布前导出结构化链接清单嵌入站点。

**技术雷达与知识周报的素材库** 技术顾问或社区运营人员可将每周阅读的技术博客、论文预印本、工具发布页统一入库，利用标签标记“待深读”“已分享”“待翻译”等状态，结合检索功能快速生成每周精选链接汇总。

**合规审计与供应链溯源** 企业法务或安全合规团队可将第三方库官网、许可证文本、漏洞通告及官方声明链接集中管理，利用健康度监控及时捕获页面变更或下线风险，为软件物料清单提供可追溯的外部证据链。

## 快速开始

以下步骤适用于 Linux / macOS / WSL2 环境，需提前安装 Git、Node.js 18.x 及以上版本与 pnpm。

```bash
# 克隆仓库
git clone https://github.com/your-org/linkvault-core.git
cd linkvault-core

# 安装依赖
pnpm install

# 初始化本地 SQLite 数据库并导入示例链接
pnpm run init-db
pnpm run seed -- --source ./data/sample-links.csv

# 启动开发服务器
pnpm run dev
```

访问控制台输出的本地地址即可开始使用 Web 界面。生产环境部署请参考 `docs/deployment.md` 中的 Docker 或 systemd 配置示例。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，推荐使用官方预编译二进制或 nvm 管理 |
| pnpm | 8.x 及以上 | 包管理与任务调度工具，需单独安装 |
| SQLite | 3.35 及以上（内置） | 默认嵌入式数据库，无需额外安装；生产环境可切换至 PostgreSQL |
| PostgreSQL（可选） | 14.x 及以上 | 用于多用户并发写入与高级全文检索，需启用 pg_trgm 扩展 |
| Redis（可选） | 6.x 及以上 | 用于分布式缓存与任务队列，提升健康度监控的调度性能 |
| Nginx / Caddy | 任意稳定版 | 推荐作为反向代理终止 TLS 并提供静态资源缓存 |

│   │   ├── src/components/            # 可复用的 UI 组件 (shadcn/ui)

│   └── cli/                           # 命令行工具 (Commander.js)
│       ├── src/commands/              # 各子命令实现 (import, check, export)
│       └── src/scheduler/             # 定时任务调度器 (node-cron)
├── packages/
│   ├── core/                          # 核心数据模型与业务逻辑 (TypeScript)
│   │   ├── src/entities/              # 链接、标签、审计日志的实体类
│   │   ├── src/repositories/          # 数据访问层 (SQLite/PostgreSQL 适配器)
│   │   └── src/validators/            # URL 规范化与元数据提取器

## 常见问题

**Q: 导入大量链接时页面出现超时或卡顿应如何处理？**

A: 建议通过 CLI 工具执行批量导入，例如 `pnpm run import -- --file ./bulk-links.csv --batch-size 200`，该命令会分批次提交并显示进度条。Web 界面上传限制为单次 500 条，超过此数量请使用命令行方式。同时可调整环境变量 `IMPORT_TIMEOUT_MS` 延长超时阈值。

**Q: 健康度检查是否会影响源站性能？是否支持自定义请求头？**

A: 检查采用 HEAD 请求，仅获取响应头信息，不会下载完整页面内容，对源站压力极小。并发请求数默认限制为 10，可通过 `CHECK_CONCURRENCY` 环境变量调整。支持在配置文件中为特定域名设置自定义 User-Agent 或 Cookie，以绕过部分站点的反爬校验。

**Q: 如何将现有浏览器书签或 Notion 数据库迁移至 LinkVault Core？**

A: 项目提供 `packages/core/src/adapters` 下的转换器，目前支持 Chrome/Firefox 导出的 HTML 书签文件、Notion 导出的 CSV 以及 Raindrop.io 的 JSON 备份。运行 `pnpm run migrate -- --adapter chrome --input ./bookmarks.html` 即可完成转换与入库。若需其他格式支持，可参考适配器接口自行扩展并提交 PR。
