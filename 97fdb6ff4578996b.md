# ResourceBridge 技术外链导航系统

ResourceBridge 是一个面向开发者、技术研究人员和内容创作者的轻量级技术资源外链汇总与导航系统。该项目定位于对分散在互联网各处的优质技术文章、百科条目、开发文档和深度分析进行结构化收录与分类索引，帮助用户快速定位特定主题下的权威参考资料，降低信息检索的时间成本。

ResourceBridge 本身不存储任何内容实体，仅提供链接元数据管理与分类导航能力，适用于个人知识库构建、团队技术文档索引、开源项目外部参考整理等场景。项目采用静态页面生成架构，支持通过 Markdown 配置文件驱动链接列表的增删改查，并内置了标签过滤与全文检索功能，便于用户在大量外链中快速定位所需信息。

## 功能概览

**多级分类导航**：支持对收录的外链按技术领域、内容类型、适用人群进行三级分类，每个链接可归属多个分类标签。

**全文检索与过滤**：内置基于倒排索引的轻量级全文检索，支持对链接标题、描述、标签进行组合查询，检索结果按相关性排序。

**元数据自动采集**：收录链接时自动发起 HEAD 请求检测资源可用性，并尝试从目标页面的 meta 标签中提取标题、描述、关键词等元信息。

**链接健康状态监控**：定时任务每日检测已收录链接的 HTTP 状态码，对返回 4xx 或 5xx 的链接进行标记并生成告警报告。

**导入导出功能**：支持 JSON、CSV、Markdown 表格三种格式的链接列表批量导入与导出，便于与其他知识管理工具进行数据交换。

**自定义主题与布局**：前端界面支持浅色/深色主题切换，列表视图与卡片视图两种展示模式，用户可根据使用习惯自由调整。

**访问统计与分析**：记录每个外链的点击次数与来源渠道，提供简单的热度排行与趋势分析，辅助用户识别高价值资源。

## 应用场景

技术团队内部知识库建设：团队可将日常开发中遇到的优质技术博客、API 参考文档、问题解决方案等外链统一收录至 ResourceBridge，按项目或技术栈分类，新成员入职时可快速了解团队常用的外部技术资源池。

开源项目外部参考整理：开源项目维护者可使用 ResourceBridge 整理项目依赖的第三方库文档、相关学术论文、社区讨论帖等外部参考链接，作为项目 README 或 Wiki 的补充索引，提升项目的可维护性与可追溯性。

技术写作者与自媒体内容素材管理：技术博主、专栏作者在撰写文章或制作视频教程时，可将参考来源、延伸阅读材料等外链通过 ResourceBridge 统一管理，便于后续引用核验与素材复用。

个人开发者学习路径规划：自学者可将不同技术领域的学习资料、官方教程、实战案例等外链按学习阶段分类收录，构建个人专属的学习资源导航，避免收藏夹杂乱无章导致的知识碎片化。

## 快速开始

以下操作基于 Linux/macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/example/resource-bridge.git
cd resource-bridge

# 安装依赖（使用 npm）
npm install

# 构建生产版本并启动服务
npm run build
npm start
```

访问 http://localhost:3000 即可进入 ResourceBridge 的导航主界面。首次启动会自动生成示例链接数据，并创建默认的管理员账户（用户名 admin，密码在控制台日志中输出，首次登录后强制修改）。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Node.js | 18.0.0 或更高 | 运行时环境，推荐使用 LTS 版本 |
| npm | 8.0.0 或更高 | 包管理器，用于安装项目依赖 |
| SQLite3 | 系统自带或手动安装 | 内置数据库引擎，用于存储链接元数据与用户配置 |
| Git | 2.25.0 或更高 | 版本控制工具，用于克隆仓库和管理配置变更 |
| 操作系统 | Linux / macOS / Windows 10+ | 跨平台支持，Windows 下建议使用 WSL2 环境 |
| 内存 | 最低 512 MB，推荐 1 GB | 运行内存要求，不包含操作系统开销 |
| 存储空间 | 最低 100 MB | 用于存放程序文件、数据库和日志，链接数量增加时存储需求相应增长 |

## 常见问题

问：ResourceBridge 是否存储外部链接的内容副本或快照？

答：ResourceBridge 仅存储链接的元数据，包括标题、描述、分类标签和最后检测时间，不存储任何外部内容的副本。所有外链点击均直接跳转至原始目标地址，项目本身不承担内容托管责任。

问：如何处理已收录链接失效的情况？

答：系统每日定时执行健康状态检测，对返回非 200 状态码的链接会在管理界面标记为异常。用户可以手动编辑链接状态，或删除失效链接。失效链接的历史访问数据仍会保留，便于追溯。

问：是否可以与现有的 Wiki 或笔记系统进行集成？

答：ResourceBridge 提供完整的 RESTful API，支持通过 API 进行链接的增删改查操作。同时支持 JSON、CSV、Markdown 表格三种格式的数据导入导出，可以与大多数笔记系统、Wiki 平台进行数据交换。具体的集成方案请参考 docs/development/API.md 文档。
