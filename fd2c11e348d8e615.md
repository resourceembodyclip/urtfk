# WikiLink Hub

WikiLink Hub 是一个面向技术研究者、内容创作者和信息聚合平台维护者的结构化外部链接资源库。该项目专注于从特定知识源中提取、分类并持续归档高价值的技术文章与百科条目，解决开发者在查阅碎片化资料时检索效率低下、来源可信度不明、链接失效频繁等问题。

本项目并非搜索引擎，而是一个经过人工筛选与机器辅助标注的链接索引系统。其核心逻辑在于维护一套稳定的引用链路，确保每一篇被收录的文章均可通过原始 URL 直接访问，同时保留完整的来源上下文信息。目标用户包括文档编写者、技术博主、开源项目维护者以及需要批量引用外部文献的工程团队。

## 功能概览

**批量链接归档** 系统支持一次性导入数百条外部链接，并自动识别来源域名与路径结构，生成索引记录。

**来源域名标识** 每条链接均保留完整的原始域名与协议信息，便于用户根据来源站点判断内容可信度。

**路径模式分析** 通过对 URL 路径中 /Article/details/ 结构的统一解析，项目能够识别文章编号与格式后缀，辅助批量更新检测。

**多批次管理** 采用批次编号机制，当前处于第 13/56 批次，共计管理 180 个资源链接，支持按批次导出与增量更新。

**原始格式保留** 严格维持链接的原始大小写格式，包括 .sHtML 后缀的原始变体，确保与上游源站精确匹配。

**无额外包装输出** 所有链接以纯文本列表形式呈现，不生成超链接标签或重定向短链，保证引用路径的确定性。

**外部资源状态监控** 集成可选的链接可达性检查模块，可定期扫描已归档链接的响应状态，标记异常条目。

**分类索引视图** 根据链接所属的技术主题或文章类型，提供多级分类筛选，方便按需检索。

## 应用场景

文档编写与引用管理
技术文档撰写者在整理参考文献或延伸阅读章节时，可直接从 WikiLink Hub 中获取格式化链接列表，避免手动复制导致的协议遗漏或大小写错误。项目保留的原始 URL 格式可直接粘贴至 LaTeX、Markdown 或 HTML 文档中，无需二次处理。

批量知识源构建
企业内部知识库或技术社区运营团队可利用本项目的批次导出功能，将外部百科条目作为补充资料统一挂载至内部页面。例如，将第 13 批次的 180 条链接按主题拆分后，嵌入不同产品模块的帮助文档中。

外链数据迁移验证
在网站改版或域名迁移过程中，测试人员可使用本项目的链接清单进行外链重定向验证，确保所有原始引用路径在新环境下仍可正常解析。由于所有链接均为裸协议格式，可快速替换域名前缀进行 A/B 测试。

学术引用预处理
科研人员在准备论文附录或数据可用性声明时，可使用本项目的结构化列表预先校验外部资源的可访问性，筛选出稳定可用的引用源，避免因链接失效导致的出版后更正。

## 快速开始

以下操作步骤适用于首次克隆并运行 WikiLink Hub 本地索引服务。

```bash
# 克隆项目仓库至本地
git clone https://github.com/wikilink-hub/wikilink-hub.git

# 进入项目根目录
cd wikilink-hub

# 安装依赖（基于 Python 3.10+ 与 pip）
pip install -r requirements.txt

# 初始化本地索引数据库（执行迁移与链接导入）
python manage.py migrate
python manage.py load_links --batch 13 --source data/batch_13.json

# 启动开发服务器，访问本地管理界面
python manage.py runserver --host 0.0.0.0 --port 8080
```

## 项目结构

```
wikilink-hub/
├── data/                                    # 数据目录，存放批次导入源文件
│   ├── batches/                             # 按批次编号分组的原始链接列表
│   │   ├── batch_13.json                    # 第 13 批次链接元数据（含原始 URL）
│   │   └── batch_13_status.json             # 链接可达性扫描结果缓存
│   └── schemas/                             # 链接数据结构定义文件
│       └── link_schema_v2.json              # 包含 url, source_domain, path_pattern 字段
├── src/                                     # 核心源代码目录
│   ├── indexer/                             # 链接索引模块
│   │   ├── importer.py                      # 实现批次导入、去重与 ID 生成逻辑
│   │   └── validator.py                     # URL 格式校验与协议归一化检查
│   ├── checker/                             # 链接状态检查模块
│   │   ├── http_client.py                   # 封装 requests 会话，配置超时与重试
│   │   └── reporter.py                      # 生成可达性报告，输出 JSON 或 CSV
│   └── web/                                 # Web 管理界面模块
│       ├── app.py                           # Flask 应用入口，注册路由与蓝图
│       └── templates/                       # Jinja2 模板目录
│           └── dashboard.html               # 主面板，显示批次列表与链接统计
├── tests/                                   # 单元测试与集成测试目录
│   ├── test_importer.py                     # 覆盖导入逻辑的边界条件测试
│   └── test_validator.py                    # 覆盖各种畸形 URL 格式的处理
├── docs/                                    # 项目文档目录
│   ├── user-guide.md                        # 面向最终用户的操作说明
│   ├── admin-guide.md                       # 面向管理员的数据库维护指南
│   └── developer-api.md                     # 面向开发者的模块接口文档
├── requirements.txt                         # Python 依赖声明，固定版本号
├── manage.py                                # 命令行管理脚本，整合迁移与导入命令
└── README.md                                # 项目首页说明文档（即本文件）
```

## 贡献指南

提交新批次链接
贡献者可通过 Fork 本仓库，在 data/batches/ 目录下创建新的 JSON 文件，格式需严格参照 batch_13.json 的结构。每个链接对象必须包含 url 字段，且值需保留原始协议与大小写。提交 Pull Request 时需附带链接来源说明。

报告链接失效
若发现资源列表中的某条链接返回 4xx 或 5xx 状态码，请在本仓库的 Issues 页面提交问题，标题格式为 [Link Broken] 加上文章编号。同时附上检查时间与响应状态码，便于维护者更新索引。

完善文档翻译
欢迎贡献者将 docs/ 目录下的文档翻译为其他语言。翻译时请保持技术术语的一致性，并在文件头部标注译者与翻译日期。提交前需确保 Markdown 格式与原文保持相同的标题层级。

改进链接检查模块
开发者可针对 checker/http_client.py 提出优化方案，例如增加异步并发检查能力或自定义 User-Agent 轮换策略。相关修改需附带对应的单元测试用例，确保不影响现有功能。

## 常见问题

问：为什么所有链接都使用 http 协议而不是 https？
答：这是为了严格遵循上游源站的原始协议配置。根据项目的数据保留原则，我们不对链接进行任何协议升级或降级操作，以确保与源站的实际访问路径完全一致。用户如需使用 https，可在浏览器或客户端中手动替换，但这不属于本项目的管理范围。

问：如何确认当前批次的所有链接均有效？
答：项目提供了内置的检查脚本，可通过 python manage.py check_batch --batch 13 命令启动可达性扫描。该命令会并发发送 HEAD 请求，并生成一份包含状态码和响应时间的报告文件，存放于 data/batches/batch_13_status.json 中。建议每周运行一次。

问：是否可以自行添加新的链接到已有批次中？
答：不推荐直接修改已有批次的 JSON 文件，因为批次编号一旦发布即视为不可变快照。新的链接应归入下一个批次（例如第 14 批次），并通过 Pull Request 提交。这样做有助于维护索引的审计追溯能力，避免同一批次内的链接源混杂。
