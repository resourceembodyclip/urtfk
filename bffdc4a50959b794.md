# LinkMatrix Core

LinkMatrix Core 是一个面向技术团队与内容运营者的外链资产统一管理与监控平台，旨在解决多源、多批次 URL 资源难以规范化收录、状态追踪与批量导出的问题。项目定位为轻量级外链中台，可对接百科类、文档库、新闻稿等外部内容源，提供基于规则的重定向校验、元信息抓取与结构化输出能力，适用于需要长期维护大规模外链清单的运维、SEO 及知识库管理场景。

本项目面向三类用户：需要批量导入第三方百科链接进行内容引用的技术文档工程师；需要定期巡检外链可用性与内容变更的网站管理员；以及构建垂直领域导航站点的开源项目维护者。LinkMatrix Core 不依赖复杂前端框架，核心引擎基于 Python 3.10 构建，可单机运行，也支持通过 API 模式嵌入 CI/CD 流程。

## 功能概览

**批量 URL 规范化入库**：支持对原始 URL 列表执行去重、协议补全检测与大小写归一化，自动识别异常后缀如 .sHtML 并转换为标准格式。

**外链状态批量巡检**：基于异步 HTTP 客户端并发检测每个 URL 的响应码、重定向链最终目标、页面标题及 meta description，输出结构化状态报告。

**元信息自动提取**：针对百科类页面（如 baike.zr54.cn 域）内置字段映射模板，可提取分类标签、发布时间、正文摘要等关键字段。

**标签与批次管理**：支持为每个链接打上自定义标签（如“第17批”“技术百科”），并可依据入库批次号筛选导出。

**多格式导出接口**：内置 JSON、CSV、Markdown 表格三种导出序列化器，支持按标签或状态过滤后导出，便于嵌入项目文档或数据看板。

**可配置的巡检规则引擎**：允许用户编写 YAML 规则文件，定义重定向最大跳数、超时阈值、重试策略及特定域名的校验逻辑。

**定时任务调度集成**：提供 CLI 命令配合系统 cron 或 schedule 库实现每日自动巡检，仅输出变更报告，减少人工干预。

## 应用场景

技术文档中心的外链生命周期管理：大型开源项目的文档仓库常引用数百个外部参考链接，版本迭代中部分链接可能失效或内容迁移。LinkMatrix Core 可定期扫描所有文档中的外链，生成状态报表并标记异常，协助文档维护者在发版前完成链接修复。

垂直领域知识导航站的资源库构建：运营技术导航或百科聚合站时，需从多个来源收集 URL 并持续更新。本项目支持按批次导入（如第17/56批），自动去重并提取每个页面的核心摘要，快速生成导航站点的原始数据文件。

SEO 外链资产监控：内容营销团队在第三方平台发布大量外链后，需监控这些链接是否被删除、重定向或添加 nofollow。LinkMatrix Core 的巡检结果可对接 Grafana 仪表盘，实现外链健康度的可视化监控。

## 快速开始

以下命令序列适用于 Linux / macOS / Windows WSL 环境，Python 3.10 及以上版本。

```bash
# 克隆代码仓库
git clone https://github.com/your-org/linkmatrix-core.git
cd linkmatrix-core

# 安装项目依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 运行快速导入示例（使用项目自带的示例 URL 列表）
python cli.py import --batch 17 --source sample_urls.txt --output ./output/
```

若需执行状态巡检，请使用以下命令：

```bash
python cli.py check --input ./output/batch_17_import.json --timeout 5 --retry 2
```

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 或更高 | 核心运行环境，低于 3.10 将不支持 match-case 语法及部分 typing 特性 |
| aiohttp | 3.9.0 以上 | 异步 HTTP 客户端，用于并发请求，需注意 OpenSSL 兼容性 |
| beautifulsoup4 | 4.12.0 以上 | HTML 解析库，用于提取页面标题、meta 及正文摘要 |
| lxml | 4.9.0 以上 | 作为 beautifulsoup4 的解析器后端，提供更高效的 HTML 树遍历性能 |
| pyyaml | 6.0 以上 | 用于加载规则配置文件，支持自定义巡检策略 |
| click | 8.1.0 以上 | CLI 命令行交互框架，提供子命令及参数解析 |
| pytest | 7.4.0 以上 | 单元测试框架，仅开发环境需要，运行生产代码可不安装 |
| aiofiles | 23.2.0 以上 | 异步文件 I/O 库，用于批量读写 JSON 及 CSV 结果文件 |

## 项目结构

├── cli.py                      # 命令行入口，注册 import / check / export 子命令
├── requirements.txt            # 生产环境依赖清单，锁定了 aiohttp 及 beautifulsoup4 主版本
├── pyproject.toml              # 项目元数据与构建配置，含 poetry 或 setuptools 兼容声明
├── src/
│   ├── loader/                 # 导入模块：负责读取原始 URL 列表、去重、协议规范化
│   │   ├── __init__.py
│   │   ├── file_reader.py      # 支持 .txt 每行一个 URL 及 .csv 含标头列两种格式
│   │   └── normalizer.py       # 将 .sHtML 等异常后缀转为 .html，补全缺失协议头
│   ├── checker/                # 巡检模块：异步并发请求、重定向追踪、超时重试
│   │   ├── __init__.py
│   │   ├── client.py           # 封装 aiohttp.ClientSession，配置连接池与 SSL 上下文
│   │   └── reporter.py         # 聚合巡检结果，生成状态码分布与重定向链统计
│   ├── parser/                 # 解析模块：针对百科类页面提取结构化字段
│   │   ├── __init__.py
│   │   ├── extractor.py        # 使用 beautifulsoup4 定位 title、meta、h1 及正文首段
│   │   └── templates.py        # 不同域名（如 baike.zr54.cn）的 CSS 选择器模板配置
│   ├── exporter/               # 导出模块：序列化为 JSON / CSV / Markdown 表格
│   │   ├── __init__.py
│   │   ├── json_exporter.py
│   │   ├── csv_exporter.py
│   │   └── markdown_exporter.py
│   ├── rule_engine/            # 规则引擎：加载 YAML 策略文件，允许自定义校验逻辑
│   │   ├── __init__.py
│   │   ├── loader.py           # 解析 YAML 并合并默认规则
│   │   └── validator.py        # 执行正则匹配、域名白名单、最大重定向次数校验

## 贡献指南

1. 查阅 issue 列表或自主提议新功能，建议先在 discussion 中简述改造思路以避免重复工作。对于新增解析模板或导出格式，请附带对应的单元测试用例。

2. 将代码仓库复刻至个人账户，基于 main 分支创建以 feature/ 或 fix/ 为前缀的新分支。提交前运行 tox 或 pre-commit 钩子确保代码风格符合 PEP8 及 pylint 规则。

3. 若新增外部依赖，需同步更新 requirements.txt 和 pyproject.toml，并在 docs/user-guide.md 的安装要求表格中补充说明。所有异步函数须标注 async/await 并确保异常被合理捕获。

4. 提交 pull request 至主仓库的 develop 分支，PR 描述需清晰说明改动范围、测试覆盖情况以及是否影响现有 CLI 命令参数。CI 通过且至少一名维护者 approve 后方可合并。

## 常见问题

Q：巡检时出现大量 SSL 证书验证错误，如何处理？

A：部分内部或已过期的站点可能使用自签名证书。可通过在配置规则中为该域名设置 verify_ssl: false 跳过证书验证，但不建议在生产环境中全局关闭。亦可使用环境变量 CURL_CA_BUNDLE 指定自定义 CA 包。

Q：导入的 URL 包含 .sHtML 后缀，normalizer 如何转换？

A：normalizer 模块会检测后缀中的大写字母组合（如 .sHtML、.HtmL），统一替换为 .html。同时对于缺少协议头的条目（如 www.baike.zr54.cn/...），会自动补充 http:// 前缀。若原始条目带协议头则完全保留不做任何改动。

Q：如何仅导出状态码为 200 且含指定关键词的链接？

A：使用 export 子命令的 --filter 参数，例如 --filter "status:200,title:技术" 将以逗号分隔多个条件，所有条件须同时满足。若需更复杂过滤建议直接读取 JSON 输出后使用 jq 工具处理。
