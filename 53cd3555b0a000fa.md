# WikiLink 聚合导航

WikiLink 是一个面向技术调研、知识管理及信息检索场景的轻量级外链聚合与导航系统。该项目不对原始内容做二次编辑或转载，仅以目录索引和结构化分类的方式，将分散于互联网各处的优质百科条目、技术文档与数据页面进行统一收录与归类，帮助开发者、研究人员及内容创作者在特定垂直领域内快速定位高价值信息源。

项目定位为一套可私有化部署的信息枢纽工具，适用于个人知识库的补充索引、团队技术决策时的参考资料池构建，以及公开数据资源的快速回溯。所有外链均以原始 URL 原样保留，项目自身不承担内容审核与可用性验证职责，但提供基础的状态标注机制供用户自行标记。

## 功能概览

**结构化索引目录** 系统根据条目来源与主题领域自动生成多级分类目录，支持按发布时间、来源域名及内容类型进行筛选，提升大规模 URL 集合下的检索效率。

**原样外链直出** 所有收录的 URL 均以原始字符串形式直接展示，不附加任何跳转中间页、跟踪参数或重定向逻辑，确保访问路径的透明性与可预期性。

**本地快照标记** 针对每个外链条目，用户可手动添加本地备注标签（如“已读”、“待验证”、“推荐”），该标记仅存储于本地配置文件，不向远程服务同步。

**批量导入导出** 支持通过纯文本列表或 CSV 格式批量导入 URL 集合，同时支持将当前索引库导出为通用书签文件（HTML 格式）或纯文本清单，便于迁移与备份。

**定期健康检查** 内置轻量级链接可用性探测模块，可按设定周期对已收录 URL 发起 HEAD 请求，并将响应状态码（2xx/4xx/5xx）记录至本地日志文件，辅助用户识别失效资源。

**全文检索补全** 基于 URL 中的路径关键词、文件名及用户自定义标签构建简易倒排索引，支持前缀模糊匹配与精确字符串查找，适用于大规模列表下的快速定位。

## 应用场景

技术选型调研 在进行框架对比、工具链评估或架构方案设计时，团队可将候选资料页、官方文档及社区评测文章统一收录至 WikiLink 索引中，形成可共享的调研素材池，避免重复搜索与链接遗漏。

学术文献补充检索 研究人员在撰写论文或综述时，可将各类百科词条、数据开放平台及机构公开报告作为背景材料纳入导航体系，配合本地标签功能标注引用优先级与可信度分级。

运维故障案例归档 运维工程师可将历史故障处理过程中参考过的外部技术贴、补丁说明及日志分析案例集中存放，形成按故障类型分类的快速参考手册，提升后续同类问题的处理效率。

个人知识库外链扩展 知识管理爱好者可在本地笔记工具之外，单独使用 WikiLink 维护一个纯外链索引库，用于存放不适合全文存储的大体积文档、动态数据面板及交互式演示页面。

## 快速开始

以下步骤适用于 Linux/macOS 及 Windows WSL 环境，需预先安装 Git 与 Node.js（版本 >= 16.0）。

```bash
# 克隆项目仓库至本地
git clone https://github.com/example/wikilink-navigator.git
cd wikilink-navigator

# 安装项目依赖（使用 npm）
npm install

# 以开发模式启动本地服务，默认监听端口 3000
npm run dev
```

启动成功后，访问控制台输出的本地地址（通常为 http://localhost:3000），即可进入索引管理界面。首次启动将自动生成示例数据与空配置文件。

├── src/                           # 核心源代码目录
│   ├── server/                    # 后端服务模块（基于 Express）
│   │   ├── index.js               # 服务入口，初始化路由与中间件
│   │   └── health.js              # 链接健康检查定时任务实现
│   ├── client/                    # 前端界面模块（原生 JavaScript + HTML）
│   │   ├── app.js                 # 主应用逻辑，渲染列表与交互控制

│   ├── storage/                   # 数据持久化层
│   │   ├── indexDb.js             # 基于 LevelDB 的键值存储封装
│   │   └── schema.js              # 数据结构定义与迁移脚本
│   └── utils/                     # 通用工具函数
│       ├── validator.js           # URL 格式校验与规范化辅助
│       └── logger.js              # 分级日志记录器（error/warn/info/debug）
├── config/                        # 配置文件目录
│   ├── default.json               # 默认配置（端口、检查间隔、分页大小）
│   └── custom.example.json        # 用户自定义配置示例文件
├── data/                          # 运行时数据目录（自动生成，不入仓）
│   ├── links.db                   # 主索引数据库文件
│   ├── tags.db                    # 本地标签存储文件
│   └── logs/                      # 应用日志及健康检查报告

## 常见问题

问：项目是否会自动修正或补全用户提供的 URL 中的协议前缀？

答：不会。WikiLink 对所有收录的 URL 采取原样存储与展示策略。用户导入时若提供裸域名（如 abc.com），系统不会自动添加 http:// 或 https://；若提供带协议或 www 前缀的字符串，系统也完全保留原始格式。项目自身不执行任何格式补全、大小写转换或尾部斜杠增删操作。建议用户导入前自行确认 URL 的可访问性。

问：健康检查模块如何判断一个链接是否失效？

答：健康检查模块通过发送 HTTP HEAD 请求获取响应状态码。若状态码属于 2xx 范围，标记为“可用”；4xx 或 5xx 标记为“异常”；超时（默认 5000 毫秒）或连接拒绝标记为“不可达”。该结果仅记录于本地日志，不自动删除或修改条目，最终处理由用户根据日志信息自行决策。

问：项目是否支持多人协作场景下的标签同步？

答：当前版本的所有用户自定义数据（标签、备注、阅读状态）均存储于本地文件系统，不提供内置的云端同步或数据库共享能力。若需在团队内共享索引库，建议将 data/ 目录中的数据库文件置于版本控制（需谨慎处理冲突）或网络共享存储中，但需注意并发写入可能引发一致性问题。
