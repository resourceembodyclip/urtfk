# LinkVault Core

LinkVault Core 是一个面向技术团队与内容运营者的外链资源归集与导航系统。该项目定位于解决分散在多平台、多文档中的外部参考链接难以统一管理、检索与共享的问题，尤其适用于需要维护大量外部参考资料的技术博客、开源项目文档站或企业内部知识库。通过结构化的链接录入、标签化分类与状态监控，LinkVault Core 帮助用户将零散的 URL 转化为可复用、可追溯的知识资产。

## 功能概览

**批量链接导入**：支持通过文本文件、CSV 或直接粘贴的方式一次性导入大量 URL，自动解析标题与元数据。

**智能状态检测**：定期对已收录的链接进行可达性检查，标记失效链接并生成报告，避免用户引用已下线的资源。

**多级标签分类**：允许用户为每条链接自定义标签体系，支持层级标签（如「编程/前端/React」），便于按维度筛选。

**全文检索与过滤**：基于链接标题、描述、标签及备注内容进行快速全文搜索，并支持按状态、日期、来源等字段组合过滤。

**链接备注与版本记录**：每条链接均可添加详细备注说明，并自动记录每次修改的版本历史，方便团队协作追溯。

**公开嵌入与只读分享**：生成只读的链接列表嵌入代码，可将特定分类或标签下的链接集合以只读方式嵌入到外部网站或文档中。

**数据导出与迁移**：支持将链接数据导出为 JSON、CSV 或 Markdown 格式，便于迁移至其他系统或进行离线备份。

## 应用场景

技术博客的外部参考管理：技术作者在撰写文章时需要引用大量官方文档、GitHub 仓库、Stack Overflow 讨论等外部资源。LinkVault Core 可为每篇文章建立独立的链接集合，并在文章发布时生成只读嵌入列表，读者可一键直达原文出处。

开源项目文档站：开源项目的 README 或贡献指南中常包含数十个外部链接（如依赖库主页、API 文档、社区论坛）。通过 LinkVault Core 集中管理这些链接，当外部资源变更或失效时，项目维护者可快速定位并更新文档中的引用。

企业内部技术周报：技术团队每周整理行业动态、工具更新、最佳实践等外链内容分享给全体成员。使用 LinkVault Core 的标签分类和版本记录功能，可轻松维护周报历史档案，新成员也能回溯过往推荐资源。

离线知识库构建：对于网络受限的开发环境，团队可预先使用 LinkVault Core 收集所有必要的外部链接，并通过批量导出功能生成完整的链接清单，配合离线缓存策略，确保封闭环境下的文档完整性。

## 快速开始

以下命令演示了如何在本地环境中克隆代码、安装依赖并启动开发服务。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/core.git linkvault-core

# 进入项目目录
cd linkvault-core

# 安装依赖（使用 npm）
npm install

# 复制环境配置文件
cp .env.example .env

# 初始化本地数据库（SQLite）
npm run db:init

# 启动开发服务器（默认端口 3000）
npm run dev
```

启动成功后，访问控制台输出的本地地址（通常为 http://localhost:3000）即可进入 LinkVault Core 的管理界面。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或更高 (LTS) | 运行时环境，建议使用 nvm 管理版本 |
| npm | 9.x 或更高 | 包管理器，用于安装项目依赖 |
| SQLite | 3.x (内置) | 默认嵌入式数据库，无需额外安装 |
| Redis | 7.x (可选) | 用于缓存与队列任务，生产环境推荐 |
| PM2 | 5.x (可选) | 进程守护工具，生产环境部署时使用 |
| Git | 2.x 或更高 | 版本控制，用于克隆与提交代码 |

## 常见问题

**问：LinkVault Core 能否处理成千上万条链接而不影响性能？**

答：可以。核心设计基于轻量级 SQLite 作为默认存储，实测可稳定管理 10 万条以内的链接记录，检索与过滤响应时间保持在 200 毫秒以内。对于更大规模的数据集，建议使用 PostgreSQL 作为生产数据库（通过修改配置中的 dialect 参数切换），并配合 Redis 缓存热点查询。

**问：链接状态检测是否会频繁请求目标服务器，导致被对方封禁？**

答：检测引擎默认采用间隔式请求策略，每条链接的检测间隔不低于 300 秒，且并发请求数可配置（默认 5 个并发）。此外支持设置 User-Agent 与请求超时阈值，模拟真实浏览器访问。对于特别敏感的域名，用户可手动将该链接加入检测白名单以跳过自动检查。

**问：如何从其他书签管理工具（如 Raindrop、Pocket）迁移数据到 LinkVault Core？**

答：项目提供了独立的迁移脚本（位于 scripts/migrate/ 目录），支持解析主流书签工具的 HTML 导出格式或 JSON 导出格式。具体操作步骤请参考文档 `/docs/user-guide/migration.md`。若需迁移其他未收录工具的数据格式，可基于 `src/services/importService.js` 中的抽象类自行扩展。
