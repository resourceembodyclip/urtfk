# LinkVault Archive

LinkVault Archive 是一个面向技术研究者、内容策展人与数据挖掘者的结构化外链资源汇总系统。本项目不生产原始内容，而是对分散于互联网各处的深度技术文章、百科条目与行业分析报告进行系统性收集、分类与索引，提供稳定可追溯的引用入口。项目定位为技术资源导航站，解决开发者在专题研究、历史资料回溯、技术决策参考时面临的信息碎片化与链接失效问题。

## 功能概览

**批量链接导入** 支持从文本文件、CSV 或直接粘贴的 URL 列表中批量导入资源，自动去重并校验可访问性。

**智能分类引擎** 基于 URL 路径模式、域名与内容特征自动推断资源所属类别，支持自定义标签体系。

**全文元数据提取** 对每个收录链接自动发起 HEAD 与 GET 请求，提取标题、内容类型、最后修改时间与响应状态码。

**结构化索引表** 将所有资源以关系型数据表形式存储，支持按日期、域名、文件扩展名、状态码等多维度排序与筛选。

**状态监控面板** 定时巡检所有已收录链接，标记失效、重定向或内容变更，生成可导出的健康报告。

**只读镜像模式** 提供轻量级静态页面生成器，将索引结果输出为纯 HTML 目录，便于内网部署或离线查阅。

## 应用场景

技术文档历史版本追踪 当开发者需要查阅某个开源库或工具的旧版本文档时，可通过本项目的日期索引快速定位对应时间点的百科快照或归档页面。

技术选型决策参考 架构师在进行中间件或框架选型时，可横向对比多个资源链接中的性能测试数据、社区活跃度与生产环境案例，形成决策依据。

离线知识库构建 企业内网用户可将本项目部署为内部知识导航页，将分散于外部站点的白皮书、规范文档与培训材料统一收录，规避外网访问限制。

数据挖掘与语料采集 自然语言处理研究人员可将本项目的链接列表作为爬虫种子源，定向采集特定域名下的结构化文本数据，用于训练或分析。

## 快速开始

以下指令适用于 Linux 与 macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/example/linkvault-archive.git
cd linkvault-archive

# 安装 Python 依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并启动索引服务
python manage.py initdb
python manage.py import --input seeds.txt
python manage.py serve --port 8080
```

种子文件 seeds.txt 每行一个 URL，本项目已内置初始资源列表，可直接使用 `--use-default-seed` 参数。

| croniter | 1.3.0 及以上 | 定时巡检任务调度表达式解析 |
| pandas | 1.5.0 及以上 | 数据导出与报表生成模块 |

提交问题报告 在 GitHub Issues 中选择 Bug Report 模板，提供完整的环境信息（操作系统、Python 版本、依赖版本）、复现步骤与预期行为。如果涉及链接失效，请附上失效 URL 与当前响应状态码。

实现新分类器 在 core/classifier.py 中扩展 CLASSIFIER_RULES 字典，添加域名或路径的正则匹配规则，并为新类别编写对应的测试用例于 tests/test_classifier.py。

完善文档 文档位于 docs/ 目录，采用 Markdown 格式。修改后执行 `python scripts/check_docs.py` 校验内部链接与锚点有效性，确保无断裂引用。

提交拉取请求 分支命名遵循 feature/xxx 或 fix/xxx 格式。提交信息采用 Conventional Commits 规范（feat:, fix:, docs:, refactor:）。拉取请求需通过 CI 所有检查项，并由至少一名维护者审核。

本地开发环境搭建 执行 `scripts/dev_setup.sh` 自动创建虚拟环境、安装依赖并初始化测试数据库。日常开发使用 `python manage.py serve --debug` 启动带热重载的 Web 服务。

## 常见问题

Q: 导入大量 URL 时出现超时或连接拒绝错误如何解决？

A: 检查网络代理设置，或在 config/default.yaml 中调大 timeout 与 retry 参数。对于批量导入，建议使用 `--batch-size 50` 参数分批次提交，避免单次请求过载。如果目标站点存在反爬机制，可在配置中设置自定义 User-Agent 与请求头。

Q: 如何迁移索引数据库到另一台服务器？

A: 直接复制 data/linkvault.db 文件即可，该数据库包含所有资源元数据与状态历史。迁移后执行 `python manage.py check --repair` 对已迁移的链接重新校验一次，确保新环境的网络访问正常。若需跨 SQLite 版本，执行 `.dump` 命令导出 SQL 脚本再导入。

Q: 状态巡检是否会触发目标站点的访问频率限制？

A: 监控模块默认采用指数退避策略，每个域名下每秒最多发起 2 个请求。可在配置中调整 per_domain_rate 与 per_ip_rate 参数。对于大型站点，建议设置 crawl_delay 为 1.0 秒以上，并在配置中排除 robots.txt 禁止的路径。
