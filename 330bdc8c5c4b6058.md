# WapWiki Directory

WapWiki Directory is a curated navigation and metadata aggregation system for mobile-accessible encyclopedia-style articles. It serves as a structured index over a large corpus of distributed knowledge entries, providing developers, researchers, and content curators with a unified access point to semi-structured article data sourced from the wap.baike.zr54.cn domain.

The project addresses the challenge of discovering and referencing specific articles within a vast, sparsely indexed collection. Instead of relying on internal site search or fragmented external caches, WapWiki Directory compiles a static, version-controlled directory of article identifiers, enabling reliable permalinking, batch processing, and integration into larger knowledge graphs or offline analysis pipelines. It is designed for technical users who require deterministic access to article content for scraping, validation, archival, or research purposes.

## 功能概览

**Stable Article Permalinks** - Provides a deterministic mapping from article IDs to their canonical mobile-accessible URLs, ensuring consistent reference across different tools and workflows.

**Bulk Metadata Index** - Maintains a machine-readable index of all catalogued articles, including their URL patterns and inferred content categories, suitable for batch processing scripts.

**Domain-Specific Query Interface** - Offers a lightweight query layer to filter articles by ID patterns, approximate publication date ranges, or content type hints extracted from the URL structure.

**Offline-Friendly Directory Snapshot** - Distributes a complete static snapshot of the article index as a JSON or CSV file, allowing users to work with the dataset without requiring live network access to the source domain.

**Integration-Ready Data Format** - Exposes the article directory via a simple RESTful API and command-line interface, with output formats including plain text, JSON Lines, and CSV for seamless integration with data pipelines.

**Versioned Change Tracking** - Records additions and removals of article entries over time, enabling users to detect content churn, monitor domain health, and audit the evolution of the indexed corpus.

## 应用场景

**Academic Research and Content Analysis** - Researchers studying the structure, evolution, or linguistic patterns of mobile encyclopedia content can use the directory to systematically sample articles, avoiding manual URL construction and reducing the risk of duplicated or omitted entries in their study samples.

**Automated Archival and Backup Pipelines** - System administrators and digital preservationists can integrate the directory into scheduled crawlers or wget scripts to maintain local mirrors of selected articles, with the index providing a clean, deduplicated list of target URLs for each archival run.

**SEO Auditing and Link Health Monitoring** - SEO specialists and webmasters can leverage the directory to periodically validate the accessibility and response codes of all listed articles, identifying broken links, redirect chains, or content removals that may affect site authority or user experience.

**Custom Search or Recommendation System Backend** - Developers building niche search engines or content recommendation systems can use the directory as a seed dataset for initial content discovery, populating their databases with a known set of articles before applying further filtering, ranking, or embedding algorithms.

## 快速开始

To quickly set up a local instance of the WapWiki Directory index and query tool, execute the following commands in a Unix-like shell environment.

```bash
# Clone the repository from the stable branch
git clone https://github.com/wapwiki/directory.git
cd directory

# Install required Python dependencies using pip and a virtual environment
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Run the initial index build to generate the directory manifest from source data
python build_index.py --input data/raw_links.txt --output data/directory.json
```

| sqlite3 (3.35 以上) | 否 | Optional dependency for users who prefer a relational storage backend for the index data. |

## 项目结构

The repository is organized to separate source data, build logic, user-facing tools, and documentation. The following ASCII tree depicts the primary directories and their purposes.

```
.
├── data/                           # Raw and processed data directories
│   ├── raw_links.txt               # Upstream source list as provided by the curator
│   ├── directory.json              # Final indexed directory output (auto-generated)
│   └── snapshots/                  # Versioned historical snapshots of the index
│       └── 2026-07-01.json         # Daily snapshot from the previous sync
├── src/                            # Source code for the index builder and utilities
│   ├── build_index.py              # Main script to parse raw links and generate JSON
│   ├── health_check.py             # Script to validate accessibility of listed URLs
│   ├── query_cli.py                # Command-line interface for searching the index
│   └── utils/                      # Helper modules for URL parsing and logging
│       ├── parsers.py              # Article ID extraction and normalization routines
│       └── validators.py           # URL scheme and format validation functions
├── tests/                          # Unit and integration tests for core modules
│   ├── test_parsers.py             # Tests for article ID extraction logic
│   └── test_validators.py          # Tests for URL validation edge cases
├── docs/                           # End-user and developer documentation
│   ├── getting-started.md          # Setup and first-run guide
│   ├── usage-guide.md              # Detailed CLI and API usage examples
│   ├── data-format.md              # Specification for the directory JSON schema
│   ├── contributing.md             # Guidelines for submitting improvements
│   └── api-reference.md            # REST API endpoint documentation (local server)
├── scripts/                        # Utility shell scripts for common tasks
│   ├── export_csv.sh               # Converts the JSON index to CSV format
│   └── sync_snapshot.sh            # Creates a timestamped snapshot of the current index
├── config/                         # Configuration files for the build system
│   └── settings.toml               # TOML file with adjustable build parameters
├── requirements.txt                # Python dependency list for pip installation
├── Makefile                        # Standard make targets for build, test, and clean
└── README.md                       # This documentation file
```

## 贡献指南

We welcome contributions that improve the accuracy, coverage, or usability of the WapWiki Directory. All contributions must adhere to the project's code of conduct and quality standards. Please follow the steps below to submit your work.

1.  **Fork the Repository and Create a Feature Branch** - Fork the main repository to your personal GitHub account, then create a new branch with a descriptive name, such as `fix-broken-links` or `add-metadata-export`, to isolate your changes.

2.  **Update the Raw Link List or Codebase** - If you are adding new article URLs, append them to `data/raw_links.txt` in the exact format as existing entries. For code changes, ensure your modifications include appropriate unit tests in the `tests/` directory and that all existing tests pass locally.

3.  **Run the Full Build and Validation Pipeline** - Execute `make build` and `make test` from the project root to regenerate the directory index and run all validation checks. Confirm that the output JSON schema remains valid and that no new warnings are introduced.

4.  **Update the Documentation if Necessary** - If your contribution changes the command-line interface, API endpoints, or output data format, update the relevant documentation files in the `docs/` folder to reflect those changes clearly.

5.  **Submit a Pull Request with a Clear Description** - Push your branch to your fork and open a pull request against the main repository's `stable` branch. Provide a concise description of the changes, the rationale behind them, and any testing results or logs that support your submission.

## 常见问题

**Q: How often is the article index updated, and how can I request a new batch of URLs to be added?**

A: The index is rebuilt and published on a weekly basis, typically every Sunday at 00:00 UTC. For addition requests, please open an issue on the GitHub repository with the subject "URL Addition Request" and include the full list of new article URLs in the exact format as they appear on the source domain. The maintainers will review and merge the additions in the next scheduled build after validation.

**Q: What should I do if I encounter a broken or inaccessible link in the directory?**

A: If you find a link that returns a 4xx or 5xx HTTP status code, or a connection timeout, please report it by filing a bug report. Include the exact URL from the directory and the observed error code. The health check script can also be run locally to validate all links, and you may submit a pull request to remove or flag problematic entries after your own validation.

**Q: Can I use this directory for commercial purposes, and what attribution is required?**

A: Yes, the directory data and code are released under the MIT license, which permits commercial use, modification, and redistribution. We request that you retain the original copyright notice and a link to this repository in any substantial redistributions of the index data, as a courtesy to the project maintainers and to ensure users can always refer back to the authoritative source.
