# EasyWiki Navigator

EasyWiki Navigator 是一个轻量级、可自托管的百科知识外链聚合与导航系统。该项目定位于技术研究者、内容创作者以及日常信息检索用户，旨在解决分散信息难以统一管理和快速访问的问题。它不直接存储内容，而是通过结构化索引将优质的外部百科条目、技术文档及知识库文章汇集于一处，提供高效、无干扰的查阅体验。

项目核心基于静态站点生成技术，所有资源链接经人工筛选与分类，覆盖计算机科学、自然科学、历史人文等多个领域。用户可通过简洁的界面按分类、标签或关键词快速定位目标文章，避免在海量信息中迷失。EasyWiki Navigator 适合作为个人或团队的知识库入口，也适用于教育场景下的参考资料整理。

## 功能概览

- **分类导航体系** 按学科领域（如编程、数学、物理、文学）对收录的百科条目进行一级分类，每个类别下再细分标签，便于用户从宏观主题逐级下钻。

- **全文关键词检索** 内置简单的倒排索引检索机制，支持对文章标题、摘要及标签字段进行快速关键词匹配，帮助用户在海量链接中精准定位相关内容。

- **外链有效性检测** 定期对收录的 URL 执行 HTTP 头检查，自动标记并隔离失效链接，确保导航列表中的条目保持可访问状态，减少死链干扰。

- **自定义收藏夹** 允许用户将常用条目标记为星标，并创建个性化的收藏分组，方便对高频访问资源进行独立管理。

- **访问统计与热门排行** 记录每条外链的被点击次数，生成热门文章排行榜，辅助用户发现当前关注度较高的知识主题。

- **响应式移动端适配** 前端界面采用流式栅格布局，在桌面浏览器、平板及手机设备上均可获得良好的阅读和操作体验。

- **数据导入导出** 支持通过 JSON 或 CSV 格式批量导入外部链接数据，同时可将当前索引库完整导出，用于备份或迁移至其他实例。

## 应用场景

- **技术团队内部知识库入口** 开发团队可将 EasyWiki Navigator 部署在内网服务器，集中整理项目相关的技术文档、API 参考及外部教程链接，新成员入职时可快速浏览并熟悉团队常用的技术资源。

- **在线教育课程参考资料整理** 教师或培训讲师可将课程涉及的拓展阅读材料（如维基百科条目、学术论文链接）录入系统，学生通过统一入口按章节或主题查找资料，减少自行搜索的盲目性。

- **个人兴趣领域信息监控** 对某个垂直领域（如机器学习、古典哲学）有持续关注需求的用户，可利用本系统分类聚合高质量外链，定期查看新增或更新的条目，形成体系化的知识积累。

- **开源文档项目的侧边导航补充** 开源软件项目可在其文档站点中嵌入 EasyWiki Navigator 的独立实例，为使用者提供项目相关的背景概念解释（如底层协议、算法原理）的外部参考链接，降低理解门槛。

## 快速开始

以下步骤指导您在本地环境中快速启动 EasyWiki Navigator 开发或生产实例。

```bash
# 克隆项目仓库至本地
git clone https://github.com/easywiki-navigator/easywiki-navigator.git

# 进入项目根目录
cd easywiki-navigator

# 安装项目依赖（使用 npm）
npm install

# 以开发模式运行，默认监听 localhost:3000
npm run dev
```

执行上述命令后，打开浏览器访问 http://localhost:3000 即可预览站点。生产环境部署请参考 `docs/deployment.md` 文档。

## 安装要求

在安装和运行 EasyWiki Navigator 之前，请确保您的环境满足以下依赖要求。

| 依赖项 | 必需版本 | 说明 |
|--------|----------|------|
| Node.js | 18.x 或 20.x LTS | 项目运行时环境，需包含 npm 或 yarn 包管理器 |
| npm | 9.x 或以上 | 用于安装项目依赖包，与 Node.js 捆绑提供 |
| SQLite3 | 3.39.x 或以上 | 默认内置数据库引擎，用于存储链接索引和用户数据，无需单独安装 |
| Git | 2.30.x 或以上 | 用于克隆仓库和管理版本更新 |
| 现代浏览器 | Chrome 100+ / Firefox 110+ / Edge 100+ | 前端界面运行环境，需支持 ES2020 及 CSS Grid 特性 |
| 磁盘空间 | 至少 200 MB | 用于存储项目源码、依赖包及数据库文件，实际需求随索引数量增长 |
| 内存 | 至少 512 MB | 开发模式运行建议值，生产环境可根据访问量调整 |

## 项目结构

项目目录遵循模块化分层设计，核心源码与配置分离，便于维护和扩展。

```
easywiki-navigator/
├── src/                            # 源代码主目录
│   ├── core/                       # 核心逻辑模块
│   │   ├── crawler.js              # 外链有效性检测与元数据抓取
│   │   ├── indexer.js              # 倒排索引构建与关键词检索
│   │   └── db.js                   # SQLite3 数据库连接与 CRUD 操作
│   ├── routes/                     # HTTP 路由层
│   │   ├── api/                    # RESTful API 端点
│   │   │   ├── links.js            # 链接增删改查接口
│   │   │   └── stats.js            # 访问统计与排行接口
│   │   └── views/                  # 页面渲染路由
│   │       ├── home.js             # 首页分类导航
│   │       └── search.js           # 搜索结果页
│   ├── public/                     # 前端静态资源
│   │   ├── css/                    # 样式表（基础 + 响应式）
│   │   ├── js/                     # 前端交互脚本（检索、收藏）
│   │   └── assets/                 # 图片及字体图标
│   └── templates/                  # EJS 模板引擎视图文件
│       ├── layout.ejs              # 全局布局模板
│       ├── index.ejs               # 首页模板
```

## 贡献指南

我们欢迎社区贡献者参与 EasyWiki Navigator 的改进。请遵循以下流程提交您的贡献。

1. 查阅问题追踪列表：访问 GitHub Issues 页面，查找未被认领且与您技能匹配的任务，或提交新问题描述您发现的缺陷或期望的功能。

2. 派生并克隆代码仓库：将主仓库派生至您的个人账户下，然后克隆至本地开发环境。建议在开发前创建一个新的功能分支，例如 `feature/your-feature-name`。

3. 编写代码并添加测试：按照项目现有的代码风格编写实现，确保新增或修改的代码包含对应的单元测试或集成测试，且所有测试用例在提交前通过。

4. 更新文档：若您的变更涉及用户可见的功能、配置项或 API 行为，请同步更新 `docs/` 目录下的相关文档，并在 `README.md` 中如有必要进行对应修改。

5. 发起拉取请求：将您的分支推送至派生仓库，然后向主仓库的 `main` 分支发起拉取请求。请在请求描述中清晰说明变更目的、实现方式及测试覆盖情况，等待维护者审核。

## 常见问题

**问题：启动时提示 SQLite3 数据库连接失败，如何解决？**

解答：首先确认系统已安装 SQLite3 动态库（Linux 下可通过 `sqlite3 --version` 检查，macOS 通常预装，Windows 需下载预编译二进制文件）。其次检查 `config/default.json` 或环境变量 `DB_PATH` 指定的数据库文件路径是否具有写入权限。若数据库文件不存在，项目首次启动时会自动调用 `config/schema.sql` 创建表结构，请确保该文件可读。若问题依旧，可尝试删除数据库文件后重新启动以触发初始化重建。

**问题：导入外部 JSON 数据时提示格式校验不通过，数据格式有何要求？**

解答：导入数据需为合法的 JSON 数组，每个元素至少包含 `title`（字符串）、`url`（字符串，需以 http:// 或 https:// 开头）、`category`（字符串）和 `tags`（字符串数组）四个字段。示例如下：`[{"title": "示例条目", "url": "http://example.com/page", "category": "计算机科学", "tags": ["算法", "数据结构"]}]`。可通过 `scripts/import.js --validate-only` 参数预先校验文件格式而不实际写入数据库。

**问题：生产环境下如何提高外链检测效率？**

解答：默认的外链检测为串行 HTTP 请求，当索引库较大时耗时较长。生产环境建议修改 `config/production.json` 中的 `checker.concurrency` 参数（例如设置为 10），启用并发检测模式。同时可配合 PM2 或 systemd 将检测任务配置为定时任务（如每日凌晨执行），避免影响白天正常访问响应速度。
