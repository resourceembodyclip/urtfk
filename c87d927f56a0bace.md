# WapWiki Resource Aggregator

WapWiki Resource Aggregator 是一个面向技术文档检索、知识库镜像与历史百科数据归档的开源外链汇总系统。该项目定位于为开发者、研究人员及内容创作者提供结构化的百科类数据入口索引，通过统一目录与分类体系，将分散在 WapWiki 平台上的大量条目进行主题化重组，降低信息发现成本。

项目不对原始内容做修改或存储，仅提供 URL 整理与分类导航服务，适用于需要快速定位特定领域条目、构建垂直知识图谱或进行批量数据采集调度的场景。通过本项目提供的索引结构与自动化脚本，用户可以高效管理超过 180 个外部百科链接，并集成至本地知识管理工具或爬虫工作流中。

## 功能概览

**批量链接结构化整理**：按照主题领域对原始 URL 进行自动归类，生成层级清晰的目录索引，支持多级标签过滤与模糊搜索。

**基础条目元数据提取**：通过 URL 模式解析条目 ID 与类型标识，提供轻量级元数据缓存机制，便于用户快速预览条目属性。

**静态索引站点生成**：内置模板引擎可将链接列表渲染为静态 HTML 导航页面，适用于内网部署或个人知识库门户搭建。

**增量更新与去重检测**：提供命令行工具检查新增链接是否与已有记录重复，支持基于条目 ID 的增量导入模式。

**导出格式多样化**：支持将整理后的链接列表导出为 JSON、CSV 或 Markdown 表格格式，便于下游数据处理工具读取。

**自定义分类规则配置**：允许用户通过 YAML 配置文件定义正则匹配规则，实现个性化的链接自动分拣逻辑。

**访问状态监测**：集成简单的 HTTP 头检测模块，可定期检查链接可用性并生成状态报告，标记失效或重定向条目。

## 应用场景

**技术文档聚合与导航**：技术团队可将本项目作为内部文档门户的链接源，将分散在各处的百科条目按技术栈或产品线分类，为团队成员提供统一的查阅入口。例如，运维团队可快速聚合与系统配置、网络协议相关的条目。

**历史数据归档与迁移辅助**：在进行网站内容迁移或归档时，本项目提供结构化的链接清单及元数据快照，帮助迁移工程师准确识别需要处理的外部依赖条目，避免遗漏关键资源。

**学术研究与引用索引**：研究人员可利用项目生成的分类索引快速定位特定领域的百科条目，作为文献综述或数据采样的起点。项目导出的 CSV 清单可直接导入文献管理软件。

**爬虫任务调度与去重**：数据采集工程师可使用项目的去重检测与导出功能，将链接清单作为爬虫任务队列的输入源，配合状态监测模块动态调整采集优先级，提高任务执行效率。

## 快速开始

以下步骤指导您在本地环境中完成项目的克隆、依赖安装与初始运行。

```bash
# 克隆项目仓库
git clone https://github.com/example/wapwiki-resource-aggregator.git
cd wapwiki-resource-aggregator

# 安装项目依赖（基于 Python 3.10+）
pip install -r requirements.txt

# 运行初始索引构建脚本，导入提供的 URL 清单
python scripts/build_index.py --input data/raw_urls.txt --output data/index.json
```

## 项目结构

项目目录采用模块化分层设计，核心代码、配置、数据与文档相互隔离，便于维护与扩展。

```
wapwiki-resource-aggregator/
├── src/                                # 核心源代码目录
│   ├── core/                           # 核心索引与数据处理模块
│   │   ├── indexer.py                  # 索引构建主逻辑，负责解析URL并生成索引记录
│   │   ├── deduper.py                  # 去重检测实现，基于条目ID进行哈希比较
│   │   └── metadata.py                 # 元数据提取与缓存管理，支持字段扩展
│   ├── exporters/                      # 导出格式实现
│   │   ├── json_exporter.py            # 导出为JSON格式，保留完整字段信息
│   │   ├── csv_exporter.py             # 导出为CSV，便于表格软件打开
│   │   └── markdown_exporter.py        # 生成Markdown表格，适用于文档嵌入
│   └── monitor/                        # 链接状态监测子模块
│       ├── checker.py                  # 发送HEAD请求检测HTTP状态码
│       └── reporter.py                 # 生成状态报告，统计有效/失效链接数量
├── config/                             # 配置文件目录
```

## 常见问题

**问：索引构建脚本报错“连接超时”，如何处理？**

答：该错误通常出现在执行状态监测模块时，由于部分目标服务器响应缓慢或网络不稳定导致。建议在 config/settings.yaml 中调大 http_timeout 参数值（默认 5 秒，可调整至 15 秒），或暂时注释掉监测相关代码，仅执行索引构建核心流程。若持续失败，可使用 --skip-monitor 选项跳过监测步骤。

**问：导出的 Markdown 表格中文显示乱码，如何解决？**

答：请确保运行导出脚本的终端环境及导出文件编辑器均使用 UTF-8 编码。在 Windows 系统中，建议使用 PowerShell 执行脚本，并在导出后使用支持 UTF-8 的编辑器（如 VS Code 或 Notepad++）打开文件。若问题依旧，可在 markdown_exporter.py 中显式指定编码参数 encoding='utf-8-sig' 以生成带 BOM 的文件。

**问：如何自定义分类规则以匹配新的 URL 模式？**

答：编辑 config/classification_rules.yaml 文件，按照已有规则格式添加新的正则表达式条目。每条规则包含 pattern 字段（正则表达式，匹配 URL 路径部分）和 category 字段（目标分类名称）。修改后重新运行 scripts/build_index.py 即可应用新规则。规则按文件顺序匹配，命中第一条后停止。
