# LinkVault Core

LinkVault Core 是一个面向技术团队与内容运营者的结构化外链资源归集与导航系统。本项目不对原始链接做任何二次封装或跳转劫持，仅提供基于分类索引与元数据标注的快速检索入口，解决海量零散链接在协作场景下的查找效率问题。目标用户包括文档工程师、技术写作者、知识库维护者以及需要定期查阅外部参考资料的研发人员。

本项目专注于链接的稳定性观测、访问状态记录与批量导入导出能力，不涉及内容抓取或存储，尊重原始资源的版权与访问策略。通过本地化的分类标签与备注字段，用户可以在不改变原始 URL 的前提下，构建属于团队自身的知识索引层。

## 功能概览

**批量链接导入** 支持从纯文本列表、CSV 文件或直接粘贴的多行 URL 数据中批量导入链接，自动去重并识别协议头。

**分类标签系统** 允许为每条链接添加多个自定义标签，支持按标签组合筛选与快速定位。

**访问状态巡检** 内置轻量级 HTTP 状态检测模块，可按计划任务模式检查链接可达性，标注异常状态。

**元数据扩展字段** 每条链接可附加备注说明、所属项目、收录日期与最后访问时间，便于追溯。

**多格式导出** 支持将当前筛选结果导出为 JSON、CSV 或纯文本列表，兼容主流文档工具与静态站点生成器。

**检索与过滤** 提供基于关键词、URL 片段、标签与状态的多条件组合检索，响应时间低于 200 毫秒。

**历史记录追踪** 记录每次链接的访问状态变化，生成简要的可用性趋势视图。

## 应用场景

技术文档团队维护外部参考链接库。文档编写过程中需要引用大量外部规范、教程与 API 参考，LinkVault Core 可为团队提供统一的链接入口，避免重复收录与失效链接混入正式文档。

开源项目 README 与官网的资源导航页生成。项目维护者可使用本系统的导出功能，将分类后的链接列表直接输出为 Markdown 格式，减少手动整理的工作量。

个人知识库的链接备份与状态监控。研究员或技术博主可将日常积累的阅读列表导入系统，定期巡检链接有效性，及时清理或更新已失效资源。

企业内部知识库的外部依赖管理。技术中台部门可使用本系统记录所有外部 API 文档、SDK 仓库与依赖库主页，当外部资源发生迁移或下线时快速响应。

## 快速开始

执行以下命令获取项目源码并启动本地服务：

```bash
git clone https://github.com/linkvault/core.git
cd core
npm install
npm run start
```

首次启动后，服务默认监听本机 3000 端口。访问 http://localhost:3000 即可进入链接管理面板。如需更改端口，可在项目根目录下创建 .env 文件并设置 PORT 环境变量。

## 常见问题

Q: 巡检任务显示超时，但链接在浏览器中可以正常打开。

A: 这是由于巡检模块默认超时时间为 3000 毫秒，且不跟随重定向。您可以在配置文件中将 CHECK_TIMEOUT 调整为 5000 或更高，并设置 CHECK_FOLLOW_REDIRECT 为 true。同时请确认目标服务器未屏蔽来自非浏览器客户端的请求头。

Q: 导入包含大量链接的 CSV 文件时页面无响应。

A: 前端界面目前单次导入上限为 500 条记录，超过此数量建议使用命令行导入模式。在项目根目录执行 npm run import -- --file=path/to/file.csv 可完成大批量导入，该命令不经过 Web 层，直接写入数据库。

Q: 导出的 Markdown 列表包含多余的反斜杠转义。

A: 这是导出模块对特殊字符的默认转义策略。您可以在导出配置中设置 escapeMarkdown: false 来禁用转义，但需注意此时输出的原始 URL 如果包含下划线或星号，在部分 Markdown 渲染器中可能被错误解析。建议保持默认转义开启。
