# LinkMaster 资源导航系统

LinkMaster 是一个面向技术社区的开源资源聚合与导航平台，专注于对分散在网络各处的技术文档、百科条目、开发教程及工程实践文章进行结构化收录与检索。项目定位为技术外链的规范化管理工具，服务于开发者、技术写作人员以及架构师群体，帮助其从海量信息中快速定位到具体技术细节、排错案例与实现参考。

本项目不生产内容，而是通过严格的URL治理机制、分类索引体系和版本追踪能力，将高质量外部资源转化为可复用、可分享、可追溯的知识索引库。当前第4批次共计收录180条技术参考链接，涵盖后端开发、数据库调优、前端工程化、运维监控、算法设计等多个领域。

## 功能概览

**多维度分类索引**：支持按技术领域、文章类型、发布时间、来源域名等维度对收录URL进行自动归类与标签化，便于按上下文检索。

**批量导入与解析**：提供脚本化工具，支持将大批量URL列表批量导入系统，并自动抓取页面标题、摘要、关键词等元数据，减少人工录入成本。

**链接健康检查**：周期性对已收录URL进行可达性检测与状态码监控，自动标记失效或重定向的链接，并提供断链报告导出功能。

**自定义标签与注释**：允许用户为每条记录添加自定义标签、阅读优先级、个人注释及关联资源引用，构建个性化知识图谱。

**Markdown格式导出**：支持将选中的资源列表按指定模板导出为结构化Markdown文档，便于嵌入技术博客、项目文档或团队Wiki。

**全文检索与过滤**：基于标题、描述、标签及URL路径关键词的轻量级全文检索引擎，支持布尔运算与正则表达式过滤。

**访问统计与热度排序**：记录每条链接的点击频次与最后访问时间，支持按热度、新鲜度或相关性排序，辅助内容筛选。

## 应用场景

技术团队内部知识库建设：团队可将日常开发中查阅的文档、排错案例、性能分析报告等外链统一录入LinkMaster，形成团队共享的技术索引，新人入职时可快速了解常用参考源，减少信息搜寻时间。

个人技术笔记与写作素材管理：技术博主或文档撰写者在创作过程中，可将参考引用的外部文章集中收录，配合标签和注释功能，在写作时快速回溯原文核心观点，确保引用准确性和版权合规。

技术选型与竞品分析调研：架构师在进行中间件选型、云服务对比或框架评估时，可将各类评测报告、性能测试数据、社区讨论帖集中收录，通过热度排序和注释对比，辅助决策过程。

开源项目外部依赖文档管理：开源项目维护者可将项目所依赖的第三方库的官方文档、API参考、迁移指南等外链纳入项目资源导航，降低贡献者的学习门槛，提升项目文档的完整性。

## 快速开始

以下步骤将指导您在本机部署LinkMaster开发环境，并完成首批资源导入。

```bash
# 克隆项目仓库
git clone https://github.com/linkmaster/linkmaster.git
cd linkmaster

# 安装项目依赖（Python 3.9+ 环境）
pip install -r requirements.txt

# 初始化本地SQLite数据库并执行迁移
python manage.py migrate

# 从CSV或TXT文件批量导入URL（示例使用sample_urls.txt）
python manage.py import_urls --file ./data/sample_urls.txt --batch 4

# 启动开发服务器
python manage.py runserver --host 0.0.0.0 --port 8080
```

访问 http://localhost:8080 即可进入Web管理界面。默认管理员账号为 admin，首次启动时请按控制台提示设置密码。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Python | 3.9 及以上 | 核心运行环境，低于3.9将导致异步IO模块兼容性问题 |
| SQLite | 3.31 及以上 | 默认内置数据库，用于存储URL元数据与索引；生产环境可切换至PostgreSQL |
| requests | 2.28.0 及以上 | 用于HTTP请求发送与元数据抓取，依赖urllib3 |
| beautifulsoup4 | 4.11.0 及以上 | HTML解析库，用于提取页面标题、描述及正文摘要 |
| lxml | 4.9.0 及以上 | 作为beautifulsoup4的解析器后端，提供高性能XML/HTML解析 |
| pytest | 7.2.0 及以上 | 单元测试与集成测试框架，仅在开发环境必需 |
| flask | 2.2.0 及以上 | Web管理界面框架，提供路由、模板及会话管理 |
| click | 8.1.0 及以上 | 命令行交互工具库，用于实现manage.py子命令 |
| schedule | 1.2.0 及以上 | 周期性任务调度库，用于链接健康检查定时任务 |
| gunicorn | 20.1.0 及以上 | 生产环境WSGI服务器，仅在部署至Linux服务器时需要 |

## 项目结构

```
linkmaster/
├── manage.py                 # 项目入口命令行工具，注册所有子命令
├── requirements.txt          # Python依赖清单，锁定所有第三方库版本
├── config/
│   ├── settings.py           # 全局配置（数据库连接、缓存、日志级别）
│   ├── urls.py               # URL路由总表，映射至各模块视图
│   └── wsgi.py               # 生产环境WSGI启动脚本
├── core/                     # 核心业务逻辑模块
│   ├── models.py             # SQLAlchemy ORM模型（URLRecord、Tag、Category）
│   ├── schemas.py            # Pydantic数据校验与序列化模式
│   ├── services/             # 业务服务层
│   │   ├── import_service.py # 批量导入与元数据抓取逻辑
│   │   ├── health_service.py # 链接健康检查与状态更新服务
│   │   └── search_service.py # 全文检索与过滤实现（基于SQLite FTS5）
│   └── utils/                # 通用工具函数
│       ├── http_client.py    # 带重试、超时、代理支持的HTTP客户端封装
│       └── parser.py         # 基于BeautifulSoup的HTML元数据抽取器
├── web/                      # Web管理界面模块（基于Flask）
│   ├── app.py                # Flask应用工厂与蓝图注册
│   ├── templates/            # Jinja2模板（列表页、详情页、导入页）
```

## 常见问题

Q: 导入大量URL时出现超时或连接被拒绝错误，应如何解决？

A: 这通常是因为目标服务器对高频请求实施了限流策略。建议使用 --delay 参数调整每次请求的间隔时间（单位秒），例如 python manage.py import_urls --file urls.txt --delay 2。同时可检查 http_client.py 中的超时配置，适当增加 connect_timeout 和 read_timeout 阈值。若问题持续，可考虑通过代理池或使用 --headless 模式仅验证URL可达性而不抓取完整页面。

Q: 如何将SQLite数据库迁移至PostgreSQL用于生产环境？

A: 首先在 config/settings.py 中将 DATABASE_URL 修改为 postgresql://user:pass@host/dbname 格式的连接字符串。然后执行 python manage.py dumpdata --format json > backup.json 导出当前SQLite数据，再执行 python manage.py migrate --database postgres 初始化PostgreSQL表结构，最后执行 python manage.py loaddata backup.json --database postgres 完成数据导入。建议在迁移前停止所有写入操作，并在测试环境中先行验证。

Q: 链接健康检查任务是否会消耗大量网络带宽或影响源站？

A: 健康检查模块默认采用单线程顺序执行，且每次请求间隔不小于1秒，对源站造成的访问压力极低。对于内部或私有网络中的链接，您可以在配置文件中设置 HEALTH_CHECK_WHITELIST 仅检查指定域名或IP范围。此外，系统支持设置请求的 User-Agent 为自定义标识，以便源站管理员识别并区分自动化流量与用户流量。
