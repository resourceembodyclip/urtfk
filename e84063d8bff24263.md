# WapWiki Indexer

WapWiki Indexer 是一个面向移动端百科类内容的高效索引与结构化整理工具。本项目定位于技术资源汇聚与外部知识库的导航汇总系统，主要服务于需要批量采集、清洗、归档移动端半结构化百科页面的数据工程师、内容运营人员以及知识图谱构建研究者。通过提供统一的资源链接管理方案，该项目解决了移动端百科条目分散、格式异构、难以批量检索与关联分析的核心痛点，帮助用户将零散的 URL 资源转化为可维护、可查询、可扩展的本地知识资产。

本项目不对原始页面内容进行二次分发，仅提供标准化的外链索引结构与本地元数据缓存框架，严格遵守互联网资源引用的基本规范。用户可通过本项目内置的模板系统快速生成适应不同来源页面的解析配置，从而将本仓库作为各类移动百科站点的事实性数据源接入层。

## 功能概览

**批量链接导入与去重** 支持从文本文件、CSV 或直接粘贴的 URL 列表中批量导入资源，自动识别重复条目并生成唯一内容指纹，避免重复索引。

**多层级标签分类系统** 内置三级标签体系，允许用户为每个链接标记主题域、内容类型与优先级，支持自定义标签扩展，便于后续按类别筛选与导出。

**元数据自动提取与补全** 对于符合预设模板的百科页面，自动从 URL 结构中解析条目编号、分类前缀及可能的版本标识，并支持通过正则表达式提取页面标题关键词。

**增量更新与变更追踪** 每次运行索引器时自动记录新增、失效或内容结构发生变化的链接，生成变更日志文件，便于用户追踪外部资源的状态漂移。

**全文检索与过滤接口** 提供基于 SQLite 全文搜索的轻量级查询引擎，支持按标签、编号范围、导入时间等多条件组合过滤，快速定位目标资源。

**结构化导出与 API 预览** 支持将索引结果导出为 JSON、CSV 或 Markdown 表格格式，同时内置简单的 HTTP 服务模块，可本机启动查看索引数据的只读 API 预览。

**链接健康检查** 集成异步 HTTP 探活功能，可按用户设定的周期自动检测已索引链接的可访问性，并标记异常状态，辅助清理无效资源。

## 应用场景

移动百科条目批量归档与本地检索
内容运营人员可定期将指定分类下的百科条目链接导入本系统，利用标签分类和全文检索功能快速筛选出特定主题的文章列表，无需反复访问原始站点进行人工翻阅，极大提升素材整理效率。

知识图谱构建的前置数据清洗环节
研究人员在构建领域知识图谱时，可利用本项目的批量导入与元数据提取功能，将大量异构的百科链接统一转换为结构化记录，再配合导出接口输出标准化数据供后续的实体链接与关系抽取流程使用。

站点迁移与内容对比辅助工具
当目标百科站点进行改版或域名调整时，运维人员可借助本项目的链接健康检查与变更追踪功能，快速识别失效链接及 URL 模式变化，为站点迁移计划提供准确的影响范围评估数据。

## 快速开始

以下指令适用于 Linux 与 macOS 环境，Windows 用户建议使用 WSL2 或 Git Bash 执行。

```bash
# 克隆仓库到本地
git clone https://github.com/your-org/wapwiki-indexer.git
cd wapwiki-indexer

# 安装依赖（建议使用 Python 3.9 及以上版本）
pip install -r requirements.txt

# 初始化数据库并运行基础索引流程
python indexer.py --input ./samples/url_list.txt --output ./data/index.db
```

首次运行完成后，索引数据默认存储在 ./data/index.db 中，可通过如下命令启动本地预览服务：

```bash
python server.py --port 8080
```

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 或更高 | 核心运行环境，低于 3.9 版本将无法支持部分类型注解与异步语法 |
| SQLite3 | 3.35.0 或更高 | 内置数据库引擎，用于存储索引记录与全文搜索表，低版本不支持 FTS5 扩展 |
| aiohttp | 3.8.0 或更高 | 异步 HTTP 客户端库，用于链接健康检查与并发请求，需保持与 Python 版本兼容 |
| beautifulsoup4 | 4.11.0 或更高 | HTML 解析辅助库，用于从页面摘要中提取纯文本信息，非核心解析路径可选 |
| lxml | 4.9.0 或更高 | 高性能 XML/HTML 解析器，作为 beautifulsoup4 的解析后端，推荐安装以提升解析速度 |
| pytest | 7.2.0 或更高 | 单元测试框架，仅开发与调试环境需要，生产部署可不安装 |

## 项目结构

```
wapwiki-indexer/
├── indexer.py                 # 主入口脚本，负责调度导入、去重与索引更新流程
├── server.py                  # 本地预览 HTTP 服务模块，基于 aiohttp 实现只读 API
├── requirements.txt           # 生产环境依赖清单，锁定主要库版本范围
├── config/
│   ├── default.yaml           # 默认配置文件，包含标签预设、超时阈值与并发数
│   └── custom.yaml.example    # 用户自定义配置模板，可覆盖默认参数
├── core/
│   ├── __init__.py
│   ├── database.py            # SQLite 数据库连接池与表结构初始化（含 FTS5 虚拟表）
│   ├── loader.py              # 链接批量加载器，支持多种输入格式解析
│   ├── deduper.py             # 基于内容指纹与精确 URL 的双重去重逻辑
│   └── extractor.py           # 元数据提取器，包含正则模板与可插拔解析器基类
├── utils/
│   ├── __init__.py
│   ├── http_client.py         # 异步 HTTP 会话封装，带重试与超时控制
│   ├── logger.py              # 统一日志模块，支持文件滚动与多级别输出
│   └── validator.py           # URL 格式校验与规范化辅助函数（仅做语法检查，不修改原始输入）
```

## 常见问题

问：索引器是否会自动补全或修正我输入的 URL 格式？
答：不会。本项目严格遵守资源列表原样输出的原则，索引器仅在内部记录时使用 URL 的字符串哈希作为主键之一，但不会对协议头、域名大小写或路径尾部斜杠做任何自动补全或修正。用户输入的 URL 会原样存储并在导出时保持原始形态。

问：如何迁移已存在的索引数据库到另一台机器？
答：直接复制 data/index.db 文件即可。该文件为独立 SQLite 单文件数据库，不依赖外部服务。若同时需要保留变更日志，请一并拷贝 data/changelog/ 目录下的历史记录文件。

问：链接健康检查功能是否会影响原始站点的正常访问？
答：健康检查模块默认使用低并发数（不超过 5 个并发请求）并设置 3 秒超时，且遵守 robots.txt 的隐式约定。用户可自行调整配置中的并发数与间隔时间，但不建议设置为过高频率，以免对目标站点造成非必要压力。
