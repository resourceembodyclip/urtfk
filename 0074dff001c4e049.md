# LinkVault Core

LinkVault Core 是一个面向技术团队与内容运营者的外链资源归集与导航系统。该项目并非一个通用的链接收藏工具，而是一套具备分类索引、状态监控与快速检索能力的轻量级外链中台。其目标用户包括技术文档撰写者、开源项目维护者、社区运营人员以及需要批量管理外部参考链接的研发团队。LinkVault Core 解决的核心问题是：当项目依赖大量外部百科、技术文章或官方文档时，如何确保这些链接的可访问性、分类的清晰性以及引用关系的可追溯性，从而降低知识碎片化带来的维护成本。

## 功能概览

- **批量链接入库与去重**：支持从文本文件、CSV 或直接粘贴的 URL 列表中批量导入链接，自动识别重复条目并生成唯一标识符。

- **分类标签与多级目录**：允许用户为每个链接分配自定义标签（Tag）及所属项目类别，支持多级目录结构，便于按技术领域、文档类型或优先级进行组织。

- **链接可用性健康检查**：内置基于 HTTP 状态码的定时检测机制，可标记失效链接（如 404、500 等），并生成不可用链接报告，辅助维护人员及时更新。

- **全文检索与过滤**：基于链接标题、描述、标签及来源域名提供轻量级全文搜索，同时支持按状态（可用/失效）、按日期范围、按分类等多维度组合过滤。

- **元数据自动补全**：对支持 Open Graph 或 Dublin Core 协议的网页，自动抓取标题（Title）、描述（Description）及最后修改时间，减少手动录入工作量。

- **外链关系导出**：支持将选定的链接列表导出为 Markdown 表格、JSON 结构或纯文本清单，便于嵌入技术文档、周报或项目汇报材料中。

## 应用场景

1. **技术文档中心的外部引用整理**  
   当技术团队维护一份涵盖数百个第三方依赖、规范文档或社区讨论帖的开发手册时，LinkVault Core 可作为外链附录的管理后台，自动检测引用链接是否仍然有效，避免文档中出现大量死链。

2. **开源项目 README 与 Wiki 的资源同步**  
   开源项目维护者可以使用本系统集中管理项目 README、贡献指南及 Wiki 页面中引用的所有外部资源链接。当链接列表发生变更时，可快速生成更新后的 Markdown 列表，直接替换原有内容。

3. **社区运营的内容编目与风险控制**  
   社区运营人员可将用户投稿、活动公告或合作伙伴官网等链接统一入库，并通过健康检查功能提前发现即将过期或已下架的页面，从而及时替换或删除无效入口。

4. **技术调研阶段的参考链路劲管理**  
   在进行竞品分析或技术选型调研时，调研人员往往需要同时跟踪数十个产品官网、性能对比报告及用户评测文章。LinkVault Core 支持按调研项目建立独立分类，并记录每条链接的收录时间与备注，方便后期回溯。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境，假设系统已安装 Git 和 Node.js（版本 >= 18.x）。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault-core.git

# 进入项目目录
cd linkvault-core

# 安装依赖（使用 npm）
npm install

# 以开发模式启动本地服务（默认监听 3000 端口）
npm run dev
```

启动成功后，访问控制台输出的本地地址（如 http://localhost:3000）即可进入 LinkVault Core 的仪表盘界面。首次启动时，系统会自动初始化一个 SQLite 数据库文件并生成示例分类数据，供快速体验。

## 贡献指南

我们欢迎并鼓励社区提交各类贡献，包括但不限于代码修复、文档改进、功能建议及新的元数据抓取器插件。请遵循以下步骤参与贡献：

1. 在 GitHub 上 Fork 本仓库，并基于 `develop` 分支创建您的特性分支（feature/your-feature-name）。请保持分支命名语义化，避免使用 `patch-1` 这类通用名称。

2. 在提交 Pull Request（PR）前，请确保所有现有单元测试与集成测试均能通过。若新增功能或修复缺陷，请同时补充对应的测试用例，覆盖率不应低于 85%。

## 常见问题

**Q：系统支持 PostgreSQL 或 MySQL 作为生产数据库吗？**  
A：默认使用 SQLite3 以降低起步门槛。生产环境中，您可以通过修改 `config/production.yaml` 中的 `db` 配置段切换至 PostgreSQL（推荐）或 MySQL。项目已在 `src/db/` 中预留了数据库适配器抽象层，您只需安装对应的驱动包并调整连接字符串即可。

**Q：健康检查会对目标站点造成较大的请求压力吗？**  
A：健康检查采用单线程顺序执行，且默认间隔为 24 小时。每次请求均设置超时（5 秒）和重试上限（1 次），并仅检查 HEAD 或 GET 请求头，不会下载完整页面内容。对于大规模链接库（超过 5000 条），建议通过环境变量 `CHECK_BATCH_SIZE` 将检查任务分批次执行。

**Q：如何将本系统与现有的技术文档生成工具（如 Docusaurus、VuePress）集成？**  
A：推荐使用导出功能。您可以在分类或标签维度下筛选所需链接，然后选择 "导出为 Markdown 列表" 格式。导出的内容可直接复制到 Docusaurus 或 VuePress 项目的 `.md` 文件中。若需自动化集成，可调用 `/api/v1/links/export` 接口，通过脚本定时拉取最新链接列表并写入文档仓库。
