# WikiLink Hub

WikiLink Hub 是一个面向技术研究者、文档维护者和知识工作者的结构化外链与百科资源聚合平台。本项目不生产内容，而是通过人工筛选与自动化校验相结合的方式，对分散在多个百科站点、技术博客和开放知识库中的高质量文章进行索引、分类与状态监控，帮助用户快速定位特定主题下的权威参考资料。

本项目定位为“技术资源导航中间层”，解决的核心问题是：当开发者需要围绕某个技术概念、历史事件或操作规范进行深入调研时，能够通过一个稳定的入口，一次性获取该主题下分布在数十个独立站点中的相关链接，而无需在多个搜索引擎间反复切换与去重。项目当前批次覆盖第 20/56 批资源，共计收录 180 个外链条目，本 README 所附列表为该批次完整清单。

## 功能概览

**多源百科条目聚合**：对来自同一主域下不同 ID 的百科文章进行统一汇聚，按内容特征自动归类至技术、历史、自然、社会等顶层目录。

**链接健康状态巡检**：每日定时检测所有收录 URL 的 HTTP 状态码，标记异常链接（4xx、5xx、超时），并在每周报告中输出变动摘要。

**语义标签自动补全**：基于文章 URL 中的数字 ID 与路径模式，结合轻量级规则引擎，为每条资源补充分类标签与关键词提示，降低人工整理成本。

**批量导入与导出**：支持通过 CSV 或纯文本列表批量新增资源，也支持将当前索引全量导出为 JSON、Markdown 或 HTML 书签格式，便于迁移或离线查阅。

**自定义视图过滤**：用户可按域名、内容类型（百科/博客/文档）、更新日期区间或自定义正则表达式过滤资源列表，快速缩小检索范围。

**只读镜像发布**：项目提供静态生成的只读镜像站点，确保在原始源站不可访问时，用户仍可查阅已缓存的资源元数据（标题、摘要、收录时间），但不缓存全文内容以规避版权风险。

## 应用场景

技术文档编写中的参考资料整理：技术撰稿人在编写系统设计文档或 API 使用指南时，需要引用多个外部来源以支撑论点。使用 WikiLink Hub 可按主题快速检索出所有相关的百科文章链接，并一键复制为标准引用格式，大幅减少手动收集时间。

开源项目 README 外链维护：开源项目维护者需要在 README 中持续更新“相关资源”或“延伸阅读”章节。通过本项目的批量导出功能，维护者可定期生成包含最新链接的 Markdown 列表，并自动过滤已失效的旧链接。

学术文献检索辅助：高校学生或研究人员在进行文献综述时，需要广泛查阅各类百科定义与背景资料。本平台提供的分类视图和标签系统可帮助其从大量杂乱的链接中筛选出与研究方向强相关的内容，提高文献调研效率。

网站迁移与链接替换审计：当某个大型百科站点变更 URL 结构或域名时，网站管理员可使用本项目的链接巡检功能，快速定位所有指向该站点的旧链接，并生成替换清单，避免外链大量破损。

## 快速开始

以下命令适用于 Linux 与 macOS 环境，Windows 用户建议使用 WSL2 或 Git Bash。

```bash
# 克隆仓库
git clone https://github.com/your-org/wikilink-hub.git
cd wikilink-hub

# 安装依赖（使用 Python 3.9+ 虚拟环境）
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化本地索引数据库并导入本批次资源
python manage.py init-db
python manage.py import-batch --batch 20 --source ./data/batch_20.txt

# 启动本地预览服务
python manage.py serve --port 8080
```

启动后，访问 http://localhost:8080 即可查看当前批次的资源列表与分类视图。

│   ├── core/                          # 核心模块
│   │   ├── indexer.py                 # 索引构建与更新逻辑

│   │   ├── import_cmd.py              # 导入批次命令

## 常见问题

**问：收录链接时是否会抓取并存储文章全文内容？**

答：不会。本项目仅存储每条 URL 的元数据，包括标题（通过 HTML title 标签获取）、收录时间、最后检查时间、HTTP 状态码以及自动生成的分类标签。项目不会缓存或存储文章正文、图片或附件，全文内容仍需用户通过原始链接访问源站获取。这一设计既规避了版权风险，也保证了源站的内容更新能够实时反映给用户。

**问：如果某个链接失效了，项目会如何处理？**

答：每日巡检服务会检测所有收录链接的 HTTP 状态码。首次检测到异常（非 200 或 3xx）时，系统会在日志中记录警告，并将该链接标记为“疑似失效”。连续三日均检测失败后，该链接将被标记为“确认失效”，并从默认视图中隐藏（用户可通过过滤选项查看全部链接）。标记为确认失效的链接会出现在每周报告摘要中，供管理员批量处理。用户也可以手动触发单条链接的实时检测，获取当前状态。

**问：我可以将自己的书签或收藏夹导入到本项目吗？**

答：可以。项目支持通过 CLI 命令导入本地书签文件（支持 Netscape HTML 书签格式和通用 CSV 格式）。导入时系统会自动去重，并尝试为每条新链接自动补充分类标签。导入后，您可以使用与现有资源相同的过滤、导出和巡检功能来管理您的个人书签集合。请注意，导入的个人书签仅存储在本地数据库中，不会自动提交至上游公共索引。
