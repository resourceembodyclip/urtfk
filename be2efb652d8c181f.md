# WapKnowledgeBridge

WapKnowledgeBridge 是一个面向移动端知识库与百科类内容聚合的开源资源导航项目。项目定位于将分散在多个移动端百科站点（如 wap.baike.zr54.cn 与 h5.baike.zr54.cn）上的结构化文章条目进行统一索引、分类整理与快速检索，为开发者、内容运营者、SEO 工程师以及自然语言处理研究人员提供可离线查阅、可二次开发的外链资源池。

本项目不直接托管文章正文，而是以高精度链接汇总表为核心，配合轻量级本地检索脚本，帮助用户在无需频繁访问源站的情况下，快速定位到所需的技术文档、历史词条、医学常识、数理公式解读、工程案例说明等深度内容。项目内所有链接均保持原始站点的真实 URI 格式，确保引用可追溯、访问路径无失真。

## 功能概览

**链接分类索引**：依据文章 ID 与来源子域名（wap 与 h5）自动将资源划分为若干逻辑分组，支持按批次、按编号范围快速筛选。

**原始 URI 保真存储**：严格保留每条链接的协议类型（http）、主机名、路径大小写（如 .sHtML 后缀）及查询参数，杜绝自动补全或格式化改写。

**本地离线检索**：内置基于 Python 的简易命令行检索工具，支持按关键词、文章 ID 区间、来源域名进行正则匹配过滤。

**元数据扩展接口**：为每条链接预留标签字段与备注字段，用户可通过外部 CSV 或 JSON 文件补充标题、摘要、分类等信息，无需修改核心代码。

**多端适配预览**：提供针对移动端浏览器优化的静态 HTML 导航页，可直接部署于 Nginx 或 GitHub Pages，作为团队内部的知识库入口。

**增量更新支持**：项目维护者可通过脚本批量追加新批次链接，自动去重并更新索引文件，适合长期运营的资源库场景。

**导出与分享**：支持将选定链接列表导出为纯文本、Markdown 表格或 JSON 格式，便于嵌入其他文档或用于模型训练数据标注。

## 应用场景

技术团队内部知识库建设。技术负责人可将本项目的链接列表作为原始素材，配合内部 Wiki 或 Confluence，快速建立涵盖多学科的外链参考库，减少重复性资料收集工作。

移动端 SEO 内容策略分析。SEO 运营人员可通过分析这批百科链接的主题分布与文章 ID 规律，判断目标站点的内容结构，为自身站点的专题策划提供数据参考。

自然语言处理语料构建。研究人员可批量获取这些链接指向的页面文本（需遵守源站 robots 协议），构建领域词典或训练轻量级文本分类模型，尤其适用于百科类短文本场景。

离线文档辅助生成。文档工程师可将这些链接作为附录资源，嵌入产品用户手册或技术白皮书，为读者提供延伸阅读路径，提升文档的权威性与完整性。

## 快速开始

以下操作步骤适用于 Linux / macOS / Windows WSL 环境，需提前安装 Git 与 Python 3.8 以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/wap-knowledge-bridge.git
cd wap-knowledge-bridge

# 安装所需依赖（使用 venv 推荐）
python3 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt

# 运行本地检索示例（按来源域名过滤）
python cli.py --source wap --output wap_links.txt
python cli.py --source h5 --output h5_links.txt
```

## 贡献指南

1. 克隆项目仓库并在本地创建功能分支，分支命名遵循 feature/xxx 或 fix/xxx 格式，确保分支基于最新的 main 分支创建。

2. 新增链接批次时，请将原始 URI 按来源域名分别追加至 data/raw/ 目录下对应的文本文件中，每行一条 URI，并注明批次编号与日期。运行 src/parser.py 进行去重校验，确保无重复条目。

3. 若需补充元数据（如文章标题、分类标签），请编辑 data/metadata/ 下对应的 CSV 文件，务必保持文章 ID 与原始链接尾部数字一致，以便脚本进行关联匹配。

## 常见问题

问：为什么资源列表中的链接带有 .sHtML 后缀且大小写不统一？这是否会影响访问？

答：这是源站点实际的 URI 格式，项目严格保留原始大小写与后缀。经测试，主流的 Web 服务器（如 Nginx、Apache）对 URI 大小写敏感程度取决于配置，但源站明确使用该格式提供服务，因此直接访问该原始 URL 可正常获取页面。项目不进行任何改写，以保证链接的可追溯性与原始性。

问：如何快速筛选出某个特定文章 ID 区间的链接？

答：使用 cli.py 的 --id-range 参数，例如 python cli.py --id-range 1000-5000 --source wap 即可输出 ID 在 1000 至 5000 之间的所有 wap 链接。该功能通过解析链接末尾的数字部分实现，不依赖外部数据库。

问：项目是否会定期检测链接的有效性（即检查是否返回 200 OK）？

答：项目提供了可选的检测模块 fetcher.py，但默认不开启自动检测，以避免对源站造成额外请求压力。维护者可手动执行 python src/fetcher.py --check-all 进行一次性检测，结果会记录在 data/availability.log 中。建议在非高峰期运行，并遵守源站的 robots.txt 规则。
